# 审计篮子独立维护 Plan

## 1. 需求文档

审计之王的搜索区负责从检查单和手册中辅助找资料；审计篮子负责人工维护最终依据。两者不再强绑定。

用户确认的新体验：

- 搜索结果只是参考，不自动写入审计篮子。
- 审计篮子是独立编辑区。
- 用户可以新增、编辑、删除条款名称。
- 每个条款下可以新增、编辑、删除多条依据内容。
- 审计篮子不绑定关键词、手册、段落位置、命中详情。
- 审计篮子支持 Excel 导入和导出，方便中断后继续维护。

业务边界不变：

- 不自动拆检查项。
- 不自动提取关键词。
- 不自动生成符合性声明。
- 不使用 AI、语义判断、向量检索或联网能力。
- 所有关键词由人添加，所有审计篮子内容由人维护。

## 6. 设计

数据模型：

- `AuditKingEvidenceGroup`
  - `id`
  - `title`
  - `items`
- `AuditKingEvidenceEntry`
  - `content`
  - `note`

状态操作：

- `addEvidenceGroup(state, title)`：新增条款组。
- `updateEvidenceGroupTitle(state, groupIndex, title)`：编辑条款名称。
- `removeEvidenceGroup(state, groupIndex)`：删除条款组。
- `addEvidenceEntry(state, groupIndex, content, note)`：新增组内依据内容。
- `updateEvidenceEntry(state, groupIndex, itemIndex, patch)`：编辑组内依据内容或备注。
- `removeEvidenceEntry(state, groupIndex, itemIndex)`：删除组内依据内容；组内删空时保留条款组，方便继续维护。
- `replaceEvidence(state, groups)`：导入时替换整个审计篮子。

导出：

- Excel 一行代表一条依据内容。
- 表头为：条款名称、依据序号、依据内容、备注。
- 同一条款多条依据输出多行。
- 空条款组也输出一行，便于保留只有名称、暂未填写内容的条款。

导入：

- 按表头读取，不依赖固定列号。
- 用条款名称归组。
- 空行跳过。

页面：

- 保留搜索区现有布局。
- 检查单参考区只保留“选中加入关键词”。
- 全部详情区只显示详情，不提供加入审计篮子按钮。
- 底部审计篮子提供新增条款、导入、导出。
- 每个条款组内提供条款名称输入框、删除条款、新增依据内容、编辑依据内容、删除依据内容。

模块边界：

- `runtime.d.ts`：更新审计篮子类型。
- `state.ts`：负责审计篮子 CRUD。
- `evidence-basket.ts`：负责审计篮子转 Excel 行。
- `export.ts`：负责 workbook 构造和按表头解析回审计篮子。
- `view.ts`：负责底部审计篮子渲染。
- `main.ts`：负责审计篮子事件接线。
- `spec/app/audit-king.md`：同步独立审计篮子事实。
