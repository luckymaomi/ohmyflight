# 校对之王性能重建 Plan

## 1. 需求文档

校对人员需要在本地比较“我的手册”和“参考手册”，优先阅读完全新增或删除的内容，再查看微调和需确认项。两份手册可以是 `.docx` 或文字型 `.pdf`。

工具必须保持可操作：比对计算不能阻塞页面；数千条差异不能令滚动、点击或文档预览失去响应。所有差异仍可查看，不能通过截断结果逃避性能问题。Word 显示接近原稿的全文并定位高亮，页面自动适配面板宽度且仅允许纵向滚动；PDF 显示原始页和命中上下文。

当前产品只服务这一套实现和“我的手册 / 参考手册”语义。重建时不保留兼容转发、旧别名、历史适配层或中间文件。

完成标准：真实手册回放期间，页面在计算、导航、定位和连续点击时保持可响应；所有差异可通过虚拟滚动访问；结构性缺失排在前面；Word/PDF 原文定位和 Excel 导出可用。

## 2. 当前事实

- 当前工作区内的校对工具改造尚未提交。
- 已确认真实回放的差异量超过两千条，直接生成全部导航卡会占用浏览器主线程。
- 已确认 Word 全文预览包含大量 DOM；每次点击重新扫描全文文本会触发布局和归一化计算。
- 已确认 PDF 页预览按点击重绘时没有页面缓存或渲染中断控制。
- 浏览器 Web Worker 是本地浏览器线程能力，不上传文档，也不需要联网。
- `docx-preview`、PDF.js、Mammoth、FlexSearch、SheetJS 已本地化，可继续作为当前实现的依赖。
- owner 已确认：PDF 不要求页内高亮；Word 只需接近原稿，不要求像素级还原。

## 3. 失败测试

- 比对计算必须在 Worker 中完成；工作台主线程只接收进度和最终结果。
- 差异导航必须保留全部条目，并且可视 DOM 数量与总差异数量无关。
- 导航排序必须是“我的手册缺失”“参考手册缺失”“微调”“需确认”。
- Word 定位索引只能在首次全文渲染后建立一次；后续点击不得重新遍历全文 DOM。
- Word 预览不得出现横向滚动条；每个渲染页必须按当前面板宽度缩放为完整可见。
- PDF 对同一页重复定位必须复用已渲染页面；新定位必须使旧异步渲染结果失效。
- 当前源文件、页面脚本、规格和测试中不得保留旧主链路文件、旧语义或兼容转发。
- Excel 必须输出完整原文、位置、匹配率、覆盖率、结构性缺失块和疑似冲突。

## 4. 目标

- 新建 Worker 比对链路，主线程在计算期间可继续响应上传界面和状态更新。
- 新建固定高度虚拟导航，只渲染当前滚动窗口附近的差异卡。
- 新建文档视图：Word 缓存全文和一次性文本索引，并按面板宽度缩放页面；PDF 缓存已渲染页并用请求编号避免过期绘制。
- 用“我的手册 / 参考手册”命名全部数据结构、页面文本、导出字段和规格。
- 删除旧工具模块和旧测试，仅保留当前实现的文件、测试和说明。

## 5. 不做范围

- 不做 OCR、扫描件 PDF、`.doc` 旧格式或联网/AI 语义判断。
- 不保证 Word/WPS 像素级分页一致。
- 不做 PDF 页内文字高亮。
- 不写回用户原始手册。
- 不提交真实手册、截图、临时报告或浏览器探测文件。

## 6. 设计

主链路：

1. 主线程读取两份手册，保留 Word 预览二进制或 PDF.js 文档句柄，并提取可传递给 Worker 的文本单元。
2. Worker 在独立浏览器线程中执行标点切片、通用噪音过滤、候选召回、评分、双向缺失识别和连续块聚合，按阶段回传进度。
3. 主线程接收当前结果，生成差异导航数据；虚拟导航按滚动位置只渲染有限数量的固定高度卡。
4. 点击导航后，Word 视图用首次建立的文本索引直接定位并高亮，渲染页自动适配面板宽度且只可纵向滚动；PDF 视图复用对应的原始页 canvas 并显示文本上下文。
5. Excel 导出直接基于 Worker 的当前结果生成五个 sheet。

模块边界：

- `manual-types.d.ts`：当前产品的数据模型。
- `manual-reader.ts`：本地读取 `.docx` / 文字型 `.pdf`，保存预览材料和文本单元。
- `comparison-core.ts`：可测试的通用切片、索引、评分、聚合和报表模型。
- `comparison-worker.ts`：Worker 协议和后台执行入口。
- `workspace-navigation.ts`：虚拟导航排序和窗口计算。
- `document-views.ts`：Word 全文缓存定位、面板宽度适配和纵向滚动，PDF 原页缓存和上下文。
- `excel-report.ts`：当前结果 Excel 导出。
- `workspace.ts`：页面状态、Worker 生命周期、事件绑定和工作台渲染。
- `main.ts`：启动入口。

状态归属：

- `myManual`、`referenceManual`、`comparison`、`selectedDifferenceId` 和 Worker 请求编号只存在工作台状态。
- Word/PDF 预览缓存只存在各自文档视图实例，不进入比对结果。
- Worker 只接收纯文本单元，不接收 File、ArrayBuffer、PDF.js 对象或 DOM。

错误与中断：

- 上传新文件或重新开始比对时终止仍在运行的 Worker。
- Worker 报错、PDF 无文字层、缺少文件或页码范围无效时显示可理解错误并允许重新上传。
- 文档定位请求使用递增编号；过期 PDF 渲染完成后不写入当前面板。

## 9. 收口

目标已完成。性能重建后新增的 Word 双侧定位索引也已完成：每份 Word 首次全文渲染后分批建立切片到 DOM 位置的映射；左侧点击微调或需确认项时，两侧 Word 从各自映射直接定位并同时高亮。结构性缺失的一侧保持原文预览但不伪造高亮。

真实回放使用 owner 提供的 Word/PDF：Worker 完成后得到 3019 条重点复核差异，其中我的手册缺失 256 块、参考手册缺失 411 块、微调 2105 条、需确认 247 条、覆盖率 61.2%。比对开始 250ms 后主线程仍可执行页面脚本。虚拟导航首屏只渲染 17 张卡、滚动到底只渲染 12 张，全部 3019 条可通过滚动访问。Word 全文整页适配面板且预览容器无横向滚动；PDF 原页 canvas 和上下文正常显示。Excel 导出包含五个当前 sheet。

验证已通过：`npm.cmd run build`、`npm.cmd run typecheck`、`npx.cmd vitest run tests/tool/proof-king`（5 个文件、9 项）和 `npm.cmd test`（42 个文件、168 项）。双 Word 浏览器回放确认两侧索引均完成，点击一条微调后我的手册有 1 个定位高亮、参考手册有 2 个候选窗口高亮。已删除仓库内真实回放临时输入、报告、摘要和探测脚本；未提交、未推送。

剩余风险：复杂 Word 图形、嵌入对象和特殊字体无法保证与 Word/WPS 像素级一致；超大手册的 Worker 计算仍需要等待，但不应阻塞页面交互。
