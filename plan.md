# 站点发布配置与 Jobskill 统一视觉 Plan

## 1. 需求文档

owner 需要只维护一个文件，就能控制工具首页图案门禁、公告、宣传页入口、贡献人员名单和每个工具入口是否显示。开关只用于隐藏页面入口，不承担权限、安全隔离或删除构建产物。

Jobskill 子站需要保持现有 Skill 搜索、左侧导航、Markdown 正文、图片和移动折叠能力，但整体视觉统一为打工皇帝工具首页的浅色页面壳、命令栏、搜索控件、边界、颜色和按钮语言。

完成标准：所有站点发布布尔开关集中在一个文件；工具开关与工具清单自动校验闭合；`false` 的入口不参与首页卡片、搜索和分类计数；Jobskill 桌面与移动端无横向溢出、导航与正文功能不变；代码、测试和文档同步。

## 2. 当前事实

已确认：`src/site-visibility.ts` 是站点入口显示开关的唯一配置源；`src/tool/tools-data.ts` 只保存工具元数据和首页公告内容，`public/sponsor/contributors.js` 只保存贡献人员及其文案。工具首页按集中配置生成卡片，构建仍会把整个 `public/` 复制到 `dist/`，因此隐藏入口不会使固定 URL 失效。

已确认：当前共有 19 个工具入口。首页另有图案门禁、公告显示和公告是否链接宣传页三个开关，贡献人员有一个发布开关。审计检查项、审计手册、培训规则等 `enabled` 属于工具内部业务状态，不属于站点发布配置。

已确认：Jobskill 使用 `public/jobskill/index.html`、`site.css`、`src/jobskill/site.ts` 和 Bootstrap，当前使用与工具首页一致的浅灰背景、熊猫品牌、搜索命令栏、中性边界和紧凑工作区。搜索、导航、Markdown 渲染、桌面双栏和移动折叠沿用同一套业务节点与逻辑。

已确认验证：构建输出 161 个脚本，类型检查、Vitest 56 文件 202 项、Python 7 项通过；桌面与 390px 浏览器路径无横向溢出或脚本错误。

## 3. 失败测试

站点配置测试必须保证：配置文件可构建；工具开关键与 19 个工具 `entry` 完全一致；所有开关为布尔值；缺失或为 `false` 时不进入首页可见集合；图案门禁、公告和贡献人员分别消费集中开关。

Jobskill 验证必须保证：六个 Skill 入口与 Markdown 路径不变；搜索和正文渲染测试继续通过；桌面与 390px 页面无横向溢出；移动菜单可展开；图片、表格、代码块和链接仍可读。

## 4. 目标

新增单一站点可见性配置，按首页、宣传页和工具入口分组。内容数据继续留在各自模块，页面渲染只读取集中开关，不创建重复状态。

Jobskill 使用与工具首页相同的浅灰背景、白色命令栏、紧凑搜索框、蓝色活动状态、细边界、6px 以内圆角和一致字体；桌面保持文档导航与正文双栏，移动保持可折叠导航。

## 5. 不做范围

不做登录、权限、服务器鉴权、私有路由或构建排除；不把业务规则开关迁入站点配置；不拆分独立仓库或独立域名；不修改 Jobskill Skill 正文、顺序、搜索算法或 Markdown 解析规则。

## 6. 设计

`src/site-visibility.ts` 是唯一发布开关源，包含首页图案门禁、公告、宣传页入口、贡献人员发布和按 `entry` 建立的工具映射。缺失键按不可见处理。`tools-data.ts` 只保留工具与公告内容，`contributors.js` 只保留贡献人员和文案。

工具页和宣传页在各自数据脚本前加载构建后的 `site-visibility.js`。类型声明明确配置结构。测试比较工具入口集合与开关键集合，避免新增工具时漏配。

Jobskill 只改 HTML 语义壳与 CSS。现有元素 ID、Bootstrap collapse 接线和 `site.ts` 生成的 `.nav-link` 契约保持不变；用现有工具首页 CSS 视觉变量重建样式，不复制业务逻辑。
