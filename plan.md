# 校对之王顺序对齐重建 Plan

## 1. 需求文档

手册修订人员需要比较“我的手册”和“参考手册”，首先发现参考手册新增但我的手册尚未包含的完整内容，其次发现参考手册已删除但我的手册仍保留的内容，再核对责任主体、数字、等级、时间、条件和普通文字修改。

页面必须让人直接查看两份原文：Word 提供可连续滚动的全文阅读和定位高亮；PDF 显示原始页面，定位到对应页但不做页内高亮。差异应按完整修订事件呈现，不能让用户处理数千个孤立片段。比对在本地完成，长时间计算和预览不能阻塞浏览器。

算法保持通用，不内置部门名、业务等级、机型、法规、标题或示例值特判。组织关系和业务等价性由人工判断，工具只忠实显示原文变化。

同时建立项目 skill，使 AI 可以脱离网页工具读取用户指定的标准 `.docx` 和文字型 `.pdf`，输出带原文证据的手册差异报告。

## 2. 当前事实

- 校对之王当前使用结构化文档模型、标点分句、唯一原文锚点、局部候选召回、顺序一对一对齐和连续修订事件聚合。
- PDF 读取按视觉位置重建行，并按跨页重复率和页面边缘位置过滤页眉页脚；Word 使用轻量全文阅读视图。
- 搜索相似度只用于候选召回；顺序、唯一占用、双侧长度和整体对应度共同约束最终修订事件。
- owner 提供的第三章样本已确认：Word 649 个段落、6 张表；PDF 61 页。独立研究确认两份文档同时存在大量一致内容、参考新增、条款扩写、责任主体变化和数字条件变化。
- 已确认参考 PDF 存在而 Word 不存在的内容包括健康管理要求、飞行训练讲评和汇报、飞行技术检查不合格处理流程、飞行技术检查复议程序等。
- owner 已确认组织主体名称变化是正常存在的文字差异，但算法不得解释或硬编码组织对应关系。
- `playwright-probe` 与 `ieb-flight-portal` 已按现有 Python 自动化和页面事实合并为 `flight-portal-probe`；其他项目 skill 职责不重叠。

## 3. 失败测试

- PDF 跨视觉行的同一句必须重建为连续文本，重复页眉页脚必须按位置和跨页频率通用过滤。
- 标点先用于切句，匹配归一化不得破坏原文、位置和逐字差异映射。
- 唯一完全一致内容必须成为顺序锚点；最终对应关系必须保持单调顺序并防止同一内容被无限复用。
- 必须支持 `1:1`、`1:N`、`N:1` 对齐，并把连续未匹配内容聚合为一个新增或删除事件。
- 参考新增、参考删除、内容修改、待确认必须可测试；不以固定业务词或样本标题决定类别。
- 页面不得在结果区一次性创建全部差异或整本 Word 的重型分页 DOM。
- 点击差异必须同时更新两侧位置：Word 高亮真实对应原文，PDF 显示真实原始页。
- 真实第三章样本结果必须能稳定识别已确认新增内容，并显著减少孤立差异数量和明显错配。

## 4. 目标

- 建立结构化文档模型，保留原文、顺序、标题、页码、表格和预览定位信息。
- 建立分层切句、精确锚点、局部候选召回、顺序对齐和修订事件聚合主链路。
- 结果以参考新增、参考删除、关键修改、普通修改和待确认排序；一致内容默认折叠。
- Word 使用轻量全文阅读视图虚拟滚动并定位高亮；原稿版式预览按需加载，不作为默认重型主链路。
- PDF 只按需渲染当前原始页并显示上下文。
- Excel 导出完整修订事件和两侧完整原文。
- `compare-manuals` skill 可独立生成证据包并指导 AI 输出差异报告。

## 5. 不做范围

- 不做 OCR、扫描件、旧 `.doc`、联网语义模型或自动业务合规结论。
- 不把组织关系、部门上下级或飞行专业规则编码为当前算法事实。
- 不做 PDF 页内高亮，不保证 Word/WPS 像素级分页还原。
- 不保留现有校对之王内部数据结构、兼容转发、旧别名或旧结果语义。
- 不提交真实手册、真实报告或临时研究文件。

## 6. 设计

主链路：文件读取 -> 结构重建 -> 分层文本单元 -> 精确锚点 -> 锚点间局部召回 -> 顺序对齐 -> 修订事件聚合 -> 虚拟化复核工作台 -> Excel 报告。

模块边界：

- `manual-model` 定义当前文档、文本单元、对应组和修订事件。
- `manual-reader` 读取 DOCX/PDF，通用识别 PDF 重复页眉页脚并重建段落。
- `alignment-core` 负责归一化、切句、唯一锚点、局部候选、顺序对齐和事件分类。
- `comparison-worker` 在 Worker 中运行纯文本对齐并报告阶段进度。
- `document-views` 提供虚拟 Word 全文和单页 PDF 原稿。
- `workspace` 管理当前文件、筛选、选择、Worker 生命周期和两侧联动。
- `excel-report` 输出当前修订事件。

状态只保存当前两份手册、对齐结果、筛选条件和选中事件。文档二进制、PDF 句柄和预览缓存不进入 Worker 结果。

上传新文件或重新比对时终止旧 Worker；预览请求使用递增编号丢弃过期结果；不可靠对应进入待确认，不强行归类。
