# 审计之王增强 Plan

## 1. 需求文档

用户已经确认审计之王的主工作流应调整为：

- 第 1 行：检查单 + 关键词。
  检查单用于人工选词，关键词池、关键词导入和关键词导出放在同一工作区。点击从检查单选出的关键词时，应能定位回检查单原条款。
- 第 2 行：两栏。
  左栏是命中摘要，显示所有手册命中的摘要列表：关键词、手册名、段落位置、前后约 100 字、精确/宽松、加入依据。
  右栏是全部详情，点击某条摘要后显示该命中附近全文，并高亮关键词。
- 底部：依据篮子。
  横向区域展示人工选中的依据，支持删除和导出 Excel。

用户还要求：

- 手册支持多次追加上传，不是每次上传都替换已有手册。
- 已上传手册可以单本删除，删除后搜索结果同步刷新。
- 关键词可以导入和导出，避免做到一半关闭页面后关键词丢失。
- 右侧摘要上下文从当前前后 20 字调整到前后约 100 字。
- 依据是“双对照”：依据名称/对应条款从检查单选中，手册依据原文从详情全文中选中，加入依据时两者绑定为同一条记录。
- 全部详情默认显示当前命中附近约 5000 字原文，不默认渲染整本手册。
- 底部依据篮子支持导入和导出，导入导出必须保留依据名称、检查单条款、手册、位置、手册原文摘录和备注。

业务边界不变：

- 不自动拆检查项。
- 不自动提取关键词。
- 不自动生成符合性声明。
- 不使用 AI、语义判断、向量检索或联网能力。
- 所有关键词由人添加，所有依据由人选择。

## 2. 当前事实

已确认代码事实：

- 审计之王页面位于 `public/tool/app/audit-king/index.html`。
- 审计之王源码位于 `src/tool/app/audit-king/`。
- 当前 `manualInput` 已设置 `multiple`，但 `handleManualFiles` 会用本次上传的 `documents` 调用 `setDocuments`，因此多次上传会替换已有手册。
- 当前手册列表只显示手册名和段落数量，没有单本删除按钮。
- 当前关键词没有记录从检查单哪个段落、哪个 offset 选出，因此点击关键词无法稳定定位回来源条款。
- 当前命中摘要上下文在 `view.ts` 中使用 `before: 20`、`after: 20`。
- 当前页面没有命中详情面板。
- 当前关键词没有导入/导出能力。
- 当前依据篮子位于底部横向区域，符合新布局方向。
- 当前依据篮子只支持导出，不支持导入。
- 当前依据记录只有关键词、手册、位置、摘录、备注，不足以表达检查单条款和手册原文的双对照关系。
- 搜索底层已经是 FlexSearch + ngram 候选召回，最终原文定位由项目代码完成。

已确认测试事实：

- `tests/tool/audit-king/` 已覆盖文本归一、搜索、高亮、状态、依据导出。
- 仓库要求修改业务规则、导出结构或解析规则时先补核心测试。

仍不明确：

- 关键词导入导出文件是否必须用 Excel。当前按项目习惯和用户说法实现为 Excel。
- 关键词导入是否要恢复检查单来源定位。当前设计：导出时保留来源字段；导入后如果检查单仍包含对应 blockId/source offset 才能继续定位，否则只恢复关键词本身。

## 6. 设计

页面信息架构：

- 第 1 行：一个大 card，内部左侧为检查单文本，右侧为关键词池和关键词导入/导出。
- 第 2 行：两栏 card，左侧为命中摘要，右侧为命中详情。
- 底部：依据篮子横向 card。

数据流：

- 上传手册 -> 追加到 `state.documents` -> 重建 documentIndex -> 重新搜索 -> 刷新手册列表、筛选器、摘要、详情。
- 删除手册 -> 从 `state.documents` 移除 -> 重建 documentIndex -> 清理手册筛选和当前详情 -> 重新搜索。
- 从检查单选词 -> 计算选区所在 blockId、start、end -> 添加关键词并保存 source -> 搜索并渲染。
- 点击关键词 -> 设置当前关键词 -> 刷新摘要 -> 如果有 source，滚动到检查单对应段落。
- 点击摘要 -> 设置当前命中索引 -> 左侧摘要激活 -> 右侧详情显示该段完整文本或附近全文。
- 从检查单选中条款 -> 写入待加入依据的名称/检查单条款。
- 从详情全文选中原文 -> 读取当前命中手册、位置、选中文本、待加入依据名称/检查单条款 -> 写入依据篮子。
- 导出关键词 -> 根据关键词池生成 Excel。
- 导入关键词 -> 读取 Excel 表头，恢复关键词池 -> 重新搜索。
- 导出依据 -> 根据依据篮子生成 Excel。
- 导入依据 -> 读取 Excel 表头，恢复依据篮子。

模块边界：

- `state.ts`：新增追加手册、删除手册、带来源添加关键词、导入关键词替换关键词池。
- `keyword-import-export.ts`：关键词 Excel workbook 构造和解析。
- `highlight.ts`：复用上下文生成能力，摘要使用前后 100 字，详情使用约 5000 字附近原文。
- `view.ts`：负责新布局渲染、来源 block 标识、摘要和详情渲染。
- `main.ts`：负责上传追加、删除、选区 offset 计算、关键词导入导出、依据名称选区、详情原文选区、摘要点击和来源滚动。
- `evidence-basket.ts`、`export.ts`：负责双对照依据行、导入和导出。
- `spec/app/audit-king.md`：同步新增手册删除、关键词导入导出、详情区和布局事实。

Excel 处理规则：

- 表头按字段名映射，不依赖固定列号。
- 导入边界把单元格值归一为字符串/布尔/数字，不把原始 cell 传到业务层。
- 不处理日期字段。
- 空关键词行跳过。
