# 审计之王手册证据绑定 Plan

## 1. 需求文档

用户要解决的实际问题：迎审时，关键词不只要能回到检查单里人工选中的来源位置，还要能人工绑定多条手册证据位置，方便后续重新上传手册后快速回到已确认过的候选依据。

谁会使用：需要对照检查单和公司手册做符合性声明的人工审核人员。

用户完成任务时应该看到的体验：选择某个关键词后，页面能显示这个关键词已绑定的手册证据；用户在命中摘要卡片上直接把某一条命中绑定或解绑为当前关键词的手册证据。全部详情只用于查看附近全文。绑定后的证据只作为定位和回看，不自动进入审计篮子。

当前范围包含：关键词状态增加一对多手册证据；页面显示证据数量和当前关键词证据列表；命中摘要卡片支持绑定/解绑当前手册命中；关键词 Excel 新增 `手册证据` sheet 导入导出；重新上传手册后按证据记录尝试恢复定位；spec、测试和构建同步。

当前范围不包含：自动判断证据是否符合条款；自动把证据写入审计篮子；把证据和审计篮子强绑定；AI、语义检索、联网检索；PDF/OCR。

业务上怎样算完成：一个关键词可以长期保存一个检查单来源和多条手册证据；导出后再导入仍能恢复关键词和证据结构；手册内容变化导致证据不能唯一恢复时，不乱猜、不影响关键词继续搜索。

## 4. 目标

用户路径完成到：用户能从命中摘要卡片把该条命中绑定或解绑为当前关键词的手册证据，并在当前关键词下看到多条证据和解绑入口。

代码主链路接到：`main.ts` 负责事件接线；`state.ts` 负责证据状态增删；定位模块负责证据恢复；`view.ts` 负责展示；`keyword-import-export.ts` 负责 Excel 两个 sheet。

状态和记录落到：每个关键词保存 `evidences` 数组；每条证据记录手册名、手册 ID、段落序号、段落 ID、起止位置、证据文本、前后文、命中模式和备注字段。

输出呈现到：关键词池显示手册证据数量；命中摘要显示绑定/解绑按钮；全部详情只显示附近全文；页面显示当前关键词手册证据列表；关键词 Excel 包含 `关键词` 和 `手册证据` 两个 sheet。

测试和文档同步到：`tests/tool/audit-king` 覆盖状态、导入导出、视图和定位；`spec/app/audit-king.md` 写清稳定性设计和禁止行为。

完成判定：局部测试、构建、类型检查、全量测试通过；代码、测试、spec 同步。

## 5. 不做范围

不做：自动选择最优证据、自动写符合性声明、自动填审计篮子、自动提取检查项和关键词。

不保留：不会把手册证据塞进关键词 sheet 同一行的隐藏拼接字段。

不兼容：不为旧的不存在手册证据结构写隐藏分支；当前导入导出以新表头和新 sheet 为事实。

不处理：PDF、OCR、`.doc`、Word 写回、跨浏览器会话自动保存。

## 6. 设计

主链路：用户选择关键词 -> 在命中摘要卡片点击绑定当前关键词 -> 状态把该条摘要对应证据加入当前关键词 `evidences` -> 证据列表刷新 -> 全部详情只用于查看附近全文 -> 导出关键词时写入第二个 sheet -> 导入关键词时恢复关键词和证据数组 -> 上传手册后尝试恢复证据定位。

输入 -> 判断 -> 状态 -> 执行 -> 输出 -> 记录：输入来自当前关键词、当前命中和 Excel；判断当前关键词不能是 `all`、当前命中必须存在；状态只写关键词证据数组；执行不修改审计篮子；输出为证据列表、高亮/定位状态和 Excel；记录为稳定文本字段和上下文。

模块边界：`state.ts` 只做状态增删改；`source-locator.ts` 增加通用位置恢复函数或手册证据恢复函数；`keyword-import-export.ts` 只管 workbook 结构；`view.ts` 只管 HTML 渲染；`main.ts` 只管事件、当前选择和提示。

文件职责：`runtime.d.ts` 定义手册证据类型；`state.ts` 管理关键词证据生命周期；`view.ts` 渲染证据数量和证据列表；`main.ts` 绑定/解绑当前命中证据；`keyword-import-export.ts` 导入导出 `手册证据` sheet；`spec/app/audit-king.md` 记录用户侧事实。

状态归属：手册证据归属关键词，不归属审计篮子；审计篮子仍是人工维护的一对多条款依据模型；检查单来源仍是关键词的单一来源定位。

数据或事件流：导入 Excel 得到关键词列表和证据列表，按关键词序号或关键词文本关联到关键词；上传/变更手册后对关键词证据尝试恢复；筛选手册不改变证据状态。

错误、恢复、中断、重试边界：没有当前关键词或当前命中时不绑定；恢复不能唯一确认时保留原证据字段并显示需人工确认；解绑只删除该关键词下指定证据；删除关键词同步删除其手册证据；删除手册不删除证据，只影响当前能否定位。

与现有架构的关系：沿用审计之王现有全局 runtime 模块结构，不引入新框架；页面继续使用 Bootstrap 卡片、按钮、滚动区域。

测试和文档影响：新增状态、导入导出、视图、定位测试；更新 spec 中关键词 Excel 结构、机械闭环、禁止行为和使用要点。
