# 校对之王真实手册最终验收 Plan

## 1. 需求文档

使用桌面上的南货航第三章 Word 和第09版第26修订第三章 PDF，对校对之王执行最终生产验收。验收不能只看自动测试通过，必须同时确认真实差异语义、算法分类、双侧原文和 PDF 页码定位基本一致。

只有不存在系统性错配、PDF 页码映射闭合、关键差异可以被稳定发现且结果适合人工复核时，才能确认校对之王本轮完成。

## 2. 当前事实

- 输入文件为 `南货航《飞行技术管理手册》R01-08 V1（无修订记录）第三章.docx` 和 `飞行技术管理手册_第09版第26修订_提取-第三章.pdf`。
- 当前实现支持 PDF 页内单元、相邻片段范围对应、近邻上下文、Word 全文定位和 PDF 连续懒加载。
- 当前完整验证为构建141个脚本、Vitest 46个文件190项通过、Python 7项通过。
- 独立证据的上一轮基线为 Word 707个结构单元/821个切片、PDF 61页/2002个结构单元/2017个切片。
- 重建后的核心在独立证据单元上输出参考新增162、参考删除22、内容修改338、待确认16；该回放不是浏览器实际文件上传结果。
- 用户要求本轮进一步执行语义审阅、真实浏览器算法回放和页码核验。
- 真实浏览器回放发现 Word 多行表格与 PDF 扁平表格无法形成范围对应，同一附件和培训课程表被同时误报为参考新增与参考删除；该问题必须通用修复后重新验收。

## 3. 失败测试

- 浏览器实际上传无法完成，Worker失败或结果无法导出时失败。
- 任一事件保存的参考原文不能在其来源PDF页的文字层中找到时失败。
- 任一事件的Word原文不能映射回其来源段落时失败。
- 内容修改两侧实质上不是同一事项，却被高置信度自动配对时失败。
- 明显新增、删除、责任主体、数字、时间或条件变化在算法结果中系统性漏失时失败。
- 新增/删除上下文锚点与事件相隔过远，导致双侧原稿不是同一区域时失败。
- 真实样本运行时间或结果规模达到无法人工使用的程度时失败。

## 4. 目标

- 获取真实浏览器上传产生的比较结果和耗时。
- 对全部事件执行来源单元与页码映射一致性检查。
- 独立语义审阅第三章关键章节和高风险修改。
- 对参考新增、参考删除、内容修改和待确认分别抽查代表事件。
- 对照独立语义证据和算法结果，记录一致、误判、漏判和不确定项。
- 确认结果达到人工复核工具标准，或继续修复后重复验收。

## 5. 不做范围

- 不让算法判断业务合规性或组织主体是否等价。
- 不要求算法替代手册修订人员作最终业务决定。
- 不把格式、页眉页脚或纯编号变化当成实质差异基准。
- 不把临时真实手册、JSON证据或浏览器输出提交到仓库。
- 不启动可见浏览器或保留本地服务。

## 6. 设计

独立语义基准使用 compare-manuals 证据提取脚本读取 Word 段落和 PDF 带页码文字块，再按章节顺序审阅锚点缺口。重点记录可从原文直接确认的新增、删除、责任主体、数字、时间、否定和条件变化。

真实算法回放使用当前 `dist` 页面、真实 File 上传、当前 ManualReader、Web Worker 和 AlignmentEvents，不能用测试夹具代替。浏览器和临时静态服务器只在验收进程中存在，结束后关闭。

位置核验基于结构化结果：事件的 `referenceUnitIds` 对应 PDF 单元页码，事件 `referenceText` 必须来自这些单元；页码文字层必须包含事件原文的稳定归一化片段。Word侧执行同样的单元归属检查。新增/删除锚点还需检查前后锚点在参考手册页码和我的手册段落顺序上邻近事件。

语义抽查优先选择高置信度修改、低置信度待确认、数字/英文标识变化、长文本、跨页内容和每类事件前中后位置样本。发现系统性错误时修改实现并重新执行完整验收。

## 8. 验证计划

运行独立证据脚本、真实页面回放脚本、来源映射审计和代表事件报告。记录运行耗时、输入单元数、切片数、事件分类数、映射失败数、远距离锚点数和语义抽查结论。

如发生代码修改，收尾运行 `npm.cmd run verify` 与 `git diff --check`。无论是否修改代码，都必须终止临时浏览器和服务器并删除临时JSON。

## 9. 收口

目标已完成。真实浏览器读取 Word 779 个原文单元、PDF 751 个原文单元，生成 766/925 个切片；读取分别耗时约 0.22/0.29 秒，比对耗时约 1.27 秒，输出参考新增72、参考删除26、内容修改227、待确认21。

全量审计确认346个事件的切片、原文单元和页码映射无失败，751个PDF单元均落在正确页；复杂表格单元在标注页的字符覆盖率为88.2%-97.1%，最佳匹配页均与标注页一致。17项关键语义差异全部命中，覆盖责任主体、数字、时间、条件、人工飞行、本场训练、检查不合格处理、复议程序和附件表格。

本轮通用修复了连续短行与扁平长文本的范围对应，消除了同一表格同时被报为新增和删除的问题；PDF跳转改为按容器内部坐标定位。第3、24、45、55、58、61页真实点击验证均命中目标原页，Word高亮单元与事件来源一致，PDF同时仅保留4-5张canvas。
