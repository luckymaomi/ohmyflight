# 有效期更新与预排逻辑白话说明

这份说明只描述 `super-training-test` 当前代码的真实行为。

## 先分清两个功能

### 一键更新有效期

它做的是：

- 读取项目 sheet 中 `培训信息是否录入 = 是` 的已录入记录
- 按项目规则计算新有效期
- 判断新有效期能不能回写到 `人员信息表`

它解决的问题是：

> 这条已经完成并录入的培训记录，是否可以把人员信息表里的有效期往后更新。

### 一键预排

它做的是：

- 读取 `人员信息表` 当前已有的有效期
- 按预排区间筛出需要安排的人
- 从同一个项目 sheet 的未录入行里提取可复用日期、默认字段和样式
- 生成预排预览和新的 `项目名预排（生成）` sheet

它解决的问题是：

> 在这次预排区间里，哪些人需要被提前安排培训。

## 当前识别结构

工具现在只认南货航这份新结构：

- `人员信息表`
- `CRM`
- 每个项目一张独立项目 sheet

项目 sheet 内部再拆成两类行：

- `培训信息是否录入 = 是`
  记为已录入记录，内部字段为 `recordedInfo`
- `培训信息是否录入 != 是`
  记为未录入记录，内部字段为 `pendingInfo`

不再存在也不再识别：

- `${项目}历史`
- `${项目}预排`

## 一键更新现在怎么做

更新时会：

1. 选定人员信息表
2. 多选项目
3. 选择这些项目共同拥有的“已录入月份”
4. 只读取该月份内的 `recordedInfo.rows`
5. 按项目规则算出新有效期
6. 只有新有效期晚于今天，且不早于旧有效期时才回写

更新明细里展示的是：

- 项目
- 项目 sheet
- 项目行号
- 旧有效期 / 新有效期
- 判断
- 处理结果

## 一键预排现在怎么做

预排时会：

1. 读取 `人员信息表` 中当前有效期
2. 按预排开始日期和结束日期筛人
3. 用项目规则判断是否进入本次预排
4. 从同一项目 sheet 的 `pendingInfo.rows` 中提取默认字段和 session 日期
5. 生成预排预览
6. 导出时新增 `项目名预排（生成）` sheet

如果未录入行里没有对应月份的日期：

- 逻辑仍然继续
- 开始/结束日期回退为用户选择的预排区间
- 备注中提示人工确认

## 当前特殊规则补充

- `危险品`
  有效期按 2 年计算；复训窗口是“失效日前 3 个日历月至失效前 1 日”。
- `英语能力 / 汉语能力`
  英语能力按 `ICAO4` 有效期 3 年，汉语能力按 `ICAO5` 有效期 6 年；考试窗口都是“失效日前 3 个日历月至失效日当天”。
- `培训开始日期`
  有效期更新按培训开始日期计算新生效锚点，不再以培训结束日期作为更新依据。
- 预排摘要里的 `30 / 60 / 90 天内到期`
  现在是按“距离本轮必须完成日期还有多少天”来分层。

## 现在最重要的结论

1. 有效期更新只依赖项目 sheet 里的已录入行。
2. 预排模板来源不再是独立 sheet，而是同一项目 sheet 里的未录入行。
3. 页面、结果明细和导出说明都不再使用“历史表 / 预排模板表”的旧概念。
4. 后续维护时，应该继续围绕 `recordedInfo` / `pendingInfo` 这套 merged sheet 结构扩展，而不是恢复旧后缀识别。
