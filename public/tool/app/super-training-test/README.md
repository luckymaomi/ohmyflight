# 超级培训维护文档

## 当前真实结构

`super-training-test` 现在只支持南货航这份新总表结构，不再兼容“历史 sheet / 预排 sheet 分开”模式。

工作簿按下面的结构识别：

- `人员信息表`
- `CRM`
- 各项目独立 sheet，例如 `应急训练`、`危险品`、`航空安保`、`飞行作风`、`英语能力`、`汉语能力`

每个项目 sheet 内同时包含两类行：

- `培训信息是否录入 = 是`
  作为已录入记录，用于有效期更新
- `培训信息是否录入 != 是`
  作为未录入记录，用于预排模板来源

## 模块职责

- `index.html`
  页面入口和静态文案。
- `scripts/config.js`
  项目规则、固定 sheet 名、默认表头和日期格式。
- `scripts/utils.js`
  日期解析、sheet 读写、样式合并、公共格式化。
- `scripts/training-record-policy.js`
  单条培训记录状态策略，统一判断已录入、取消和异常。
- `scripts/scanner.js`
  识别人员信息表和项目 sheet，并把项目行拆成 `recordedInfo` / `pendingInfo`。
- `scripts/validity.js`
  只基于 `recordedInfo` 更新 `人员信息表` 中的有效期。
- `scripts/schedule.js`
  基于 `人员信息表` 现有有效期筛人，并从 `pendingInfo` 提取默认值、日期和复用样式所需上下文。
- `scripts/schedule-assessment.js`
  统一排班评估核心，结合人员信息表、项目 sheet、规则核心和记录策略输出排班状态、筛选项、统计卡片和图表数据。
- `scripts/workbench.js`
  排班总览薄封装，只调用 `schedule-assessment.js` 并提供筛选后的视图。
- `scripts/workbench-export.js`
  导出当前排班总览视图。
- `scripts/expiry-list.js`
  只读查询人员信息表中指定月份范围内到期的人员项目记录。
- `scripts/expiry-list-export.js`
  导出到期清单查询结果。
- `scripts/plan-check.js`
  按月份核对当月到期人员是否已有可覆盖本轮到期的安排，并直接在原 sheet 上做绿名/补加红名。
- `scripts/rule-reference.js`
  基于 `config.js` 和 `rule-engine.js` 渲染规则速查，不维护独立算法。
- `scripts/report-sheet.js`
  生成最前面的 `更新报告` sheet。
- `scripts/generated-sheet.js`
  生成新的 `项目名预排（生成）` sheet。
- `scripts/app.js`
  页面交互、预览和导出。

## 识别口径

- 人员信息表优先识别 `人员信息表`，否则按必需表头兜底识别。
- 项目 sheet 只按项目同名 sheet 或别名识别，不再拼接 `历史` / `预排` 后缀。
- `CRM` 继续忽略。
- 月份来源：
  - 有效期更新使用 `recordedMonths`
  - 预排上下文使用 `pendingMonths`

## 一键更新

- 从所选项目的 `recordedInfo.rows` 中筛选所选月份。
- 只处理 `培训信息是否录入 = 是` 的行。
- 按现有规则计算新有效期。
- 只回写 `人员信息表` 对应项目列。
- 明细里展示来源 `项目 sheet` 和 `项目行号`，不再展示“历史 sheet”概念。

## 一键预排

- 筛人来源是统一排班评估核心。
- 只生成 `已过期`、`必须排`、`推荐排` 三类人员。
- `已排未录入`、`已录入待更新`、`正常` 不重复生成预排。
- 预排模板来源是同一项目 sheet 的 `pendingInfo.rows`。
- 从未录入行提取：
  - 可用月份
  - 默认字段
  - session 日期
  - 表头结构
  - 可复用样式
- 如果某个月没有可复用日期，就回退到用户选择的预排区间，不因为样式缺失阻塞功能。

## 排班总览

- 扫描范围是 `人员信息表` 中所有人员和所有可识别项目。
- 默认评估日期区间是当天到下月底。
- 可以在页面上手动修改评估开始日期和评估结束日期。
- `已过期`、`必须排`、`推荐排` 均以评估日期区间为锚点判断。
- 默认视图只显示需要排班处理或人工确认的状态：
  - 已过期
  - 必须排
  - 推荐排
  - 异常
- 状态下拉可查看其他状态：
  - 已排未录入
  - 已录入待更新
  - 正常
- 已排判断复用 `RuleEngine.evaluatePlanCoverage`。
- 推荐排和必须排由 `schedule-assessment.js` 统一判断，默认区间是当天到本月底。
- 有窗口项目进入窗口后显示为 `推荐排`，到期月仍未排显示为 `必须排`。
- 无窗口项目提前 1 个月进入 `推荐排`，到期月仍未排显示为 `必须排`。
- 如果可覆盖记录里没有 `培训信息是否录入 = 是`，显示为 `已排未录入`。
- 如果已录入记录可覆盖本轮到期，但人员信息表仍是旧有效期，显示为 `已录入待更新`。
- 支持按项目、状态、到期月份和关键字筛选。
- 页面默认只展示简洁列：状态、项目、姓名、当前有效期、已排日期、说明。
- 导出 Excel 保留完整列：状态、项目、员工号、姓名、当前有效期、到期月份、最晚完成日期、已排日期、是否录入、来源、说明。
- 排班总览扫描和筛选不修改 Excel；导出只生成排班总览结果副本。

## 到期清单查询

- 只读取 `人员信息表` 中的当前有效期。
- 按培训类型、开始月份和结束月份筛选到期人员。
- 开始月份等于结束月份时，只查询该月份。
- 开始月份早于结束月份时，查询完整月份范围。
- 不读取项目 sheet 里的安排记录。
- 不判断是否已排。
- 不修改原表。
- 有效期无法解析的记录进入异常提示。

## 培训计划核对

- 筛人来源仍是 `人员信息表` 当前有效期。
- 口径按月处理：只要某人的有效期落在所选月份，就进入本轮核对。
- “已排”的定义是：
  - 对应项目 sheet 中存在同人记录。
  - 该记录的培训日期按项目规则能覆盖本轮到期。
- 备注包含 `取消` 的未录入记录不算已排覆盖。
- `培训信息是否录入 = 是` 且备注包含 `取消` 的记录按异常提示。
- “缺失”的处理是：
  - 直接在原项目 sheet 末尾补一行。
  - 只填 `姓名` 和 `有效期`。
  - 已排人员姓名标绿，补加人员姓名标红。

## 维护注意

- 后续如果南货航继续沿用这类 merged sheet，只需要维护表头差异和项目规则。
- 如果 `培训信息是否录入` 或备注取消口径变化，需要优先修改 `training-record-policy.js`。
- 不要再引入 `${项目}历史`、`${项目}预排`、`historySheetName` 这一类旧结构字段。
