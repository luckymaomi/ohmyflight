# 超级培训

`super-training-test` 当前只支持南货航新总表结构，不再兼容“历史 sheet / 预排 sheet 分开”模式。

## 当前可见入口

- 排班总览
- 有效期更新

页面不再提供独立的到期清单查询、预排班设置和培训计划核对入口。

## 排班总览

- 导入总培训表后自动扫描。
- 默认评估日期区间是当天到下月底。
- 可以手动改评估开始日期和评估结束日期。
- 默认只显示需要关注的状态：已过期、已过期已排补训、必须排、已排未覆盖、推荐排、异常。
- 已排未录入、已录入待更新、正常可以通过状态下拉查看。
- 支持按项目、状态、到期月份、人员关键字筛选。
- 支持导出当前筛选总览。
- 支持点击项目风险矩阵后导出右侧人员明细。

## 有效期更新

- 只看已录入记录。
- 选择项目和共同已录入月份后生成预览。
- 导出时生成 Excel 副本并添加更新报告。
- 不覆盖原文件。

## 总表识别

- 人员信息表：当前有效期来源。
- 项目 sheet：已录入和未录入培训记录来源。
- 项目 sheet 只按项目同名 sheet 或别名识别，不再拼接 `历史` / `预排` 后缀。

## 记录过滤

- 备注包含 `取消` 的未录入记录不参与覆盖判断。
- `培训信息是否录入 = 是` 且备注包含 `取消` 的记录按异常提示。
- 自动过滤清单维护在 `training-ignore-list.ts`。

## 维护约束

- 培训规则以 `spec/app/super-training-test/training-rule-spec.md` 为准。
- UI 只展示结果，不承载业务判断。
- 排班总览、有效期更新、覆盖判断共用核心规则。
- 行为变化时必须同步代码、测试和规格。
