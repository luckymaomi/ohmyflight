规则口径

这里统一说明 training-tools-suite 当前采用的规则口径，已与 super-training-test 的判定方式保持一致。

基准月

项目：应急训练。

首次培训先确定基准月。若复训命中基准月前后 1 个月窗口，则基准月不变，直接顺延到下一轮；未命中窗口则按本次培训月份重建基准月。

3个月窗口

项目：英语能力、汉语能力。

窗口从失效日前 3 个日历月开始，直到失效日当天结束。英语能力按 ICAO4 顺延 3 年，汉语能力按 ICAO5 顺延 6 年。

3个月窗口（前一日截止）

项目：危险品。

窗口同样从失效日前 3 个日历月开始，但必须在失效前 1 日前完成。若在窗口内完成，则仍沿用旧到期锚点顺延 2 年。

最新日期

项目：航空安保、疲劳管理、飞行作风。

这一类没有窗口期，也没有保护锚点。只认最新一次培训开始日期，直接按有效期重算。

案例说明

下面用例子把命中窗口、超期、提前或窗口外三类情况说明白。

数据规范

把导入要求、处理口径和导出内容集中写清楚，便于填表和排查问题。

培训预排班页

  • 导入 1：规则表文件，优先识别名为 规则表 的工作表；未命名时按规则字段自动匹配。
  • 导入 2：有效期表文件，默认读取第一张工作表。
  • 筛选口径：使用 预排开始日期 + 预排结束日期 的区间。有窗口项目看区间是否与窗口重叠，无窗口项目看旧有效期是否落在本阶段内。
  • 导出结果：摘要、已过期、命中窗口、本阶段到期、缺少旧有效期、未命中窗口 / 阶段外未到期。

培训有效期更新页

  • 培训文件内需要同时包含 规则表 和 培训表，也就是“规则表 + 某个培训表”的组合文件。
  • 有效期表默认读取第一张工作表，至少包含 员工号 / 姓名 以及待更新的项目列。
  • 参与计算口径：只有 培训信息是否录入 = 是 的记录才参与更新；更新结果区分为已更新、不变、有效期回退、更新无效、跳过。
  • 导出结果：摘要、更新预览、跳过记录、更新后有效期表；成功回写的有效期单元格会高亮。

培训冲突核对页

  • 只需导入 1 个表，至少包含 姓名 / 项目名称 / 培训开始日期 / 培训结束日期 4 列。
  • 如果导入的是整本培训工作簿，工具会优先识别名为 培训表 的工作表；若未命名，也会按表头自动匹配。
  • 核对口径：同名人员只要培训日期区间有重叠，就判定为冲突；首尾同日也算冲突。
  • 导出结果会保留原表列，并将冲突行整行标红，便于后续复核。

关键字段规范

  • 规则表字段至少要有：项目名称、规则类型、有效期数值、有效期单位、基准月浮动月数、到期取整方式、是否启用。
  • 培训表字段至少要有：员工号、姓名、项目名称、培训开始日期、培训结束日期、培训信息是否录入。
  • 培训冲突核对表至少要有：姓名、项目名称、培训开始日期、培训结束日期。
  • 有效期表至少要有：员工号、姓名，以及项目对应列。
  • 日期建议直接使用 Excel 日期单元格；若填文本，支持 yyyy/mm/dd 和 yyyy-mm-dd。没有旧有效期时留空，不适用可写“ 不适用 ”。

模板下载

培训表模板内包含规则表、培训表和填写说明三张工作表；有效期表模板是当前项目的标准宽表结构。

  • 规则表里的 有效期数值 / 基准月浮动月数 建议保持数值格式，不要设成日期。
  • 培训表模板按最少必填字段收口，真正参与有效期计算的是 培训开始日期，培训结束日期用于留档和核对。
  • 如果直接导入整本“培训表模板（含规则表）”文件，培训冲突核对会自动识别其中的 培训表 工作表。
  • 有效期表项目列建议直接填 Excel 日期；若填文本，优先使用 yyyy/mm/dd，也兼容 yyyy-mm-dd；没有旧值就留空，不适用可写“不适用”。
  • 工具导出的日期列会优先写成 Excel 日期单元格，默认显示为 yyyy/mm/dd。
