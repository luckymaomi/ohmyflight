# 锁班皇帝用户手册

## 解决什么问题

锁班皇帝把批量锁班记录自动录入飞行门户的非生产任务页面，减少逐人填写员工号、类型、日期和备注的重复工作，并把成功、冲突和错误实时写入结果文件。

## 使用前准备

- 安装 Python、Playwright、openpyxl 等脚本所需依赖，并安装 Chromium。
- 从工具页下载 `app.py`；大量数据且接受实验性并发时可下载 `superapp.py`。
- 下载并填写锁班 Excel 模板，或准备格式一致的粘贴文本。
- 逐条检查员工号、姓名、锁班类型、开始日期和结束日期；准备好本轮白名单和可选统一备注。

## 怎样操作

1. 常规任务运行 `python app.py`；大量任务需要并发时运行 `python superapp.py`。
2. 按终端提示设置浏览器、白名单和统一备注。统一备注确认后会用于本轮全部记录。
3. 在打开的浏览器中人工登录飞行门户，并进入非生产任务录入页面。
4. 选择批量粘贴、手动单条或 Excel 导入方式，确认脚本识别到的记录数和白名单过滤结果。
5. 开始处理后观察浏览器和终端，不要手动切换脚本正在操作的页面。
6. 完成或中断后打开新生成的结果 Excel，逐条核对成功、冲突和错误记录。
7. 串行版如生成失败记录文本，只整理失败项后再重跑；已成功项不要重复提交。

## 怎样理解结果

- 成功表示门户已完成当前记录对应的提交动作。
- 冲突表示门户返回现有任务或排班冲突，脚本不会强行覆盖。
- 错误表示输入、页面或提交过程未完成，备注中会保留原因。

## 操作注意事项

- 这是会写入生产系统的自动化，运行前必须人工复核输入和白名单。
- `app.py` 是稳定串行入口；`superapp.py` 并发更快，但更需要核对结果归属。
- 结果 Excel 每条实时保存；异常退出后先以结果文件为准，排除成功项再恢复。
- Excel 原文件不会被回写，结果在新文件中。
- 门户页面升级、弹窗或登录失效时应停止并检查，不能把空结果当成功。
