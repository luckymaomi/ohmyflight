# 校对之王用户手册

## 解决什么问题

校对之王用于比较同一本手册的两个版本，集中找出参考手册新增、参考手册删除、双方内容修改和自动判断不稳的待确认项。它把每项差异放回两侧完整原文中，帮助修订人员逐项复核，不替代业务判断。

## 使用前准备

- 明确哪份是“我的手册”，哪份是“参考手册”；方向会直接决定新增和删除的含义。
- 两份文件可为标准 `.docx` 或带文字层的 `.pdf`。
- 如果 PDF 只需要比较部分正文，提前确认正确页码范围。
- 扫描件、图片 PDF、旧 `.doc` 当前不支持。

## 怎样操作

1. 分别上传“我的手册（基准）”和“参考手册（对照）”。PDF 可填写读取页码范围；当前方向会显示在页面状态区。
2. 等待两侧完成读取，再开始比对。大手册需要较长时间，不要在处理中刷新页面。
3. 先看总览和左侧大类计数，再选择 `参考新增`、`参考删除`、`内容修改` 或 `待确认`。大类和搜索决定当前完整事件流，章节树用于快速定位。
4. 展开一个章节后点击小节，事件流会滚动到该小节起点，不会隐藏其他小节事件。点击事件后阅读顶部两侧命中原文和前后上下文；颜色只表示逐字差异。
5. 在下方 Word 阅读区或 PDF 原页视图核对真实位置。新增或删除的一侧没有对应原文时，页面会定位到附近共同内容。
6. 使用大类和关键词继续缩小当前待办范围。搜索后的大类计数以 `命中数/总数` 显示，章节和小节只保留包含命中事件的组；关键词结果会标明命中侧并显示命中片段。
7. 对每个事件选择 `待处理`、`纳入报告` 或 `不纳入`。可把当前大类与搜索范围批量纳入，也可只看已纳入事件。
8. Excel 和 Word 均可选择导出全部事件或仅导出已纳入事件。Word 的“对比说明”列保持空白，供后续人工填写。
9. 需要换电脑、浏览器或继续工作时保存项目 ZIP；下次直接导入项目 ZIP，会同时恢复两本原手册、比较结果、人工决定和当前视图。

## 怎样理解结果

- `参考新增`：参考手册有，而我的手册在相同顺序位置没有可靠对应。
- `参考删除`：我的手册有，而参考手册在相同顺序位置没有可靠对应。
- `内容修改`：两侧内容能够对应，但文字发生增删或替换。
- `待确认`：系统找到相邻候选，但证据不足以可靠定类。
- 一致内容只进入统计，不显示为修订待办。
- 章节和小节只是既有修订事件的阅读分组和滚动锚点，不会改变算法判断，也不会把事件流过滤成单个小节。相邻且属于同一稳定小节的事件在报告中归在一起，但每个事件仍保留自己的类别、位置、原文和判断依据。

## 操作注意事项

- 上传方向错了会让新增、删除含义完全反转。
- 自动对应不等于业务等价；数字、否定词、责任主体和适用范围必须人工读原文。
- PDF 页面只显示原页，不做页内文字高亮；应结合顶部原文和页码核对。
- 已配置的独立噪音整行会在 PDF 读取时忽略；包含相同文字的正常句子不会被删除。
- 新增或删除位于文档开头/结尾，或前后没有可靠共同锚点时，页面会明确显示原因，不代表原页漏渲染。
- 待确认项不能直接忽略，它通常表示表格、列表或排版变化使自动对应不稳定。
- 无法可靠识别标题的事件会进入 `未识别章节` 或 `未识别小节`，不代表内容被忽略。
- Word 使用章节、小节、对比说明、我的手册、参考手册五列表格；对比说明留空，同一呈现组会合并章节和小节单元格，但子事件仍逐行保留。我的手册删除文字为红色，参考手册新增文字为蓝色，相同文字为黑色。
- Excel 不合并单元格，每个事件仍占一行，并附带人工决定、章节、小节和组内序号，便于继续筛选和排序。
- 导出的 Excel 和 Word 是复核证据，不会改写任何原手册。
- 项目 ZIP 才是完整恢复文件；单独的 Excel 和 Word 只用于复核和交付，不能恢复原手册与页面状态。
