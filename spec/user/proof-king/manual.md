# 校对之王用户手册

## 解决什么问题

校对之王用于比较同一本手册的两个版本，集中找出参考手册新增、参考手册删除、双方内容修改和自动判断不稳的待确认项。它把每项差异放回两侧完整原文中，帮助修订人员逐项复核，不替代业务判断。

## 使用前准备

- 明确哪份是“我的手册”，哪份是“参考手册”；方向会直接决定新增和删除的含义。
- 两份文件可为标准 `.docx` 或带文字层的 `.pdf`。
- 如果 PDF 只需要比较部分正文，提前确认正确页码范围。
- 扫描件、图片 PDF、旧 `.doc` 当前不支持。

## 怎样操作

1. 分别上传“我的手册（基准）”和“参考手册（对照）”。PDF 可填写读取页码范围；当前方向会显示在页面状态区。
2. 等待两侧完成读取，再开始比对。大手册需要较长时间，不要在处理中刷新页面。
3. 先看总览中的四类数量，再从 `参考新增` 开始逐项复核，然后查看 `参考删除`、`内容修改` 和 `待确认`。
4. 点击某个修订事件，阅读顶部两侧命中原文和前后上下文；颜色只表示逐字差异。
5. 在下方 Word 阅读区或 PDF 原页视图核对真实位置。新增或删除的一侧没有对应原文时，页面会定位到附近共同内容。
6. 使用类别、章节和关键词筛选缩小当前待办范围。关键词结果会标明命中侧并显示命中片段。
7. 需要完整数据和继续筛选时导出 Excel；需要接近人工对照稿的阅读版时导出 Word。

## 怎样理解结果

- `参考新增`：参考手册有，而我的手册在相同顺序位置没有可靠对应。
- `参考删除`：我的手册有，而参考手册在相同顺序位置没有可靠对应。
- `内容修改`：两侧内容能够对应，但文字发生增删或替换。
- `待确认`：系统找到相邻候选，但证据不足以可靠定类。
- 一致内容只进入统计，不显示为修订待办。

## 操作注意事项

- 上传方向错了会让新增、删除含义完全反转。
- 自动对应不等于业务等价；数字、否定词、责任主体和适用范围必须人工读原文。
- PDF 页面只显示原页，不做页内文字高亮；应结合顶部原文和页码核对。
- 已配置的独立噪音整行会在 PDF 读取时忽略；包含相同文字的正常句子不会被删除。
- 新增或删除位于文档开头/结尾，或前后没有可靠共同锚点时，页面会明确显示原因，不代表原页漏渲染。
- 待确认项不能直接忽略，它通常表示表格、列表或排版变化使自动对应不稳定。
- Word 使用五列表格，对比说明列留空；我的手册删除文字为红色，参考手册新增文字为蓝色，相同文字为黑色。
- 导出的 Excel 和 Word 是复核证据，不会改写任何原手册。
