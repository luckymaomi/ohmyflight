# ohmyflight 开源宣传页

工具首页是否提供宣传页入口由 `src/site-visibility.ts` 中的 `homepage.sponsorEntry` 控制：`true` 启用入口，`false` 只保留公告原文。

页面首先显示“为什么是 ohmyflight？”、项目定位、GitHub 仓库入口、真实案例入口和项目徽标。随后以左侧八荣、中央“龙颜大悦”、右侧八耻的结构展示“八荣八耻”，说明项目以真实业务需求、可用结果、事实复现、业务测试、诚实边界、减少重复劳动、生产力归还给人和开放共建为荣，并反对制造虚假问题、表面文章、夸大宣传、只求能跑、万能工具话术、为炫技增加流程、效率提高就多接活以及拒绝质疑。

“真实问题，正在运行的工具”依次展示猫咪、批量锁班、起落数查询、自动点 OA 和培训工作台的现有样例演示。校对之王位于案例之后，用纯文字说明唯一锚点、最长递增子序列、滑动窗口、2-gram 相似度和最大权重链，以及参考新增、参考删除、内容修改和待确认四类结果。

页面不提供具体业务工具跳转。猫咪部分显示官网和安装命令；三个命令右侧均有复制按钮。OA 助手只自动处理可直接确认的“已阅”待办，需要审批、填写意见、签署或特殊处理的事项仍由用户完成。

页面末尾通过贡献指南、GitHub Issues 和 Pull requests 提供参与路径。反馈问题应说明使用场景、输入样例、期望结果和复现步骤；提交代码或文档改进时应附带验证结果。

“贡献人员”区域用于记录提供真实场景、大量测试、代码、文档或其他重要帮助的参与者。只填写姓名和贡献说明即可；链接是可选项且不限 GitHub，留空时姓名和头像不可点击。头像可以使用本地图片或网络图片，没有头像时显示统一匿名头像。名单统一在 `public/sponsor/contributors.js` 中维护，是否显示名单由 `src/site-visibility.ts` 中的 `sponsorPage.contributors` 控制。

贡献人员区域的标题、等待文字、空状态说明和人数格式也在 `contributors.js` 的 `labels` 中维护；`countTemplate` 里的 `{count}` 会替换为当前发布人数。
