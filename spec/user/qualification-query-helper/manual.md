# 技术等级运行资格查询助手用户手册

## 解决什么问题

这个助手按 Excel 员工号逐人查询 IEB 技术资料页面，完整收集技术等级和运行资格两张表，并把每个人的处理状态实时写入结果文件。它适合批量盘点资质，不会修改门户数据。

## 使用前准备

- 下载 `app.py`、`requirements.txt` 和 Excel 模板。
- 安装依赖：`pip install -r requirements.txt`，再运行 `playwright install chromium`。
- 模板必填 `员工号` 或 `工号`，可填 `姓名`；员工号必须是六位数字。
- 去掉不需要查询的人，保留原输入文件备查。

## 怎样操作

1. 运行 `python app.py`。
2. 在浏览器中扫码登录 IEB，依次进入“资质管理 / 飞行训练 / 技术资料 / 资料管理”。
3. 等脚本确认页面就绪后，在终端输入 Excel 路径。
4. 核对读取到的有效人数、重复员工号和输入错误数量，再开始查询。
5. 运行期间不要切换脚本正在使用的标签页。每完成一人，结果会立即保存。
6. 完成或中断后先看结果 Excel 的 `汇总`，再看 `处理报告` 中的失败和姓名不一致项。
7. 需要查看具体资格时，在 `技术资料明细` 按员工号和资料类型筛选。

## 怎样理解结果

- `处理报告` 每个输入人员一行，用于判断是否查询成功、页面姓名是否一致和两类记录数量。
- `技术资料明细` 一条技术等级或运行资格记录一行，是具体业务数据。
- `汇总` 记录成功、失败、输入错误和中断状态。
- 同名文本报告便于快速查看本轮异常。

## 操作注意事项

- 查询以员工号为唯一定位；输入姓名只做核对。
- 重复员工号只查询第一条，其他重复行会写进处理报告。
- 标签加载失败会自动重试一次，仍失败的人员需要人工复核。
- 中断后已完成人员仍在结果文件中；重新运行前先排除已完成项可节省时间。
- 门户页面结构变化时不要继续相信空表，应根据失败状态检查脚本。
