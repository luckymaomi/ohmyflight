# 审计之王用户手册

## 解决什么问题

审计之王用于把检查单中的每个检查项与公司手册原文连接起来，形成可以复核的审计证据链。它能在多本 Word/PDF 手册中按关键词找原文、保存候选证据、整理最终审计依据，并进一步定位到 PDF 页面导出证据。

## 使用前准备

- 一份标准 `.docx` 检查单。
- 一本或多本标准 `.docx` 或带文字层的 `.pdf` 手册。
- 每个检查项需要人工确认编号、名称和一个搜索关键词。
- 扫描件和图片型 PDF 没有可读文字层，当前不能使用。

## 怎样操作

1. 上传检查单和公司手册，确认文件均显示读取成功。
2. 新建检查项，分别填写检查编号、检查项名称和关键词。也可以在检查单原文选中文字，建立或更新检查项来源。
3. 启用需要检索的检查项和手册，执行搜索。
4. 打开命中结果，核对完整原文和上下文。合适的内容先保存为“手册证据”。
5. 对确认可用的候选点击“采纳为审计依据”；需要补充时可直接新增、编辑或删除人工依据。
6. 随时导出检查项工作簿保存进度。下次导入该工作簿会恢复整套检查项、候选和依据。
7. 依据确认后进入 PDF 证据工作区，选择对应 PDF，查看系统建议页码。
8. 人工调整页码和前后扩页，预览无误后导出 PDF 证据或保存工作区 JSON。

## 怎样理解结果

- 搜索命中只是关键词在原文中出现，不代表它已经足以支持检查结论。
- “手册证据”是待确认候选；“审计依据”才是用户决定纳入交付的内容。
- PDF 自动定位只给连续页候选，覆盖不足时会保持未命中，必须人工确认。

## 操作注意事项

- 编号、名称和关键词互不自动推断，导入后发现空值应人工补齐。
- 不要直接把搜索摘要当结论，应打开上下文并核对章节、适用对象和例外。
- 检查项工作簿必须包含三个规定 sheet；传统审计篮子结果表不能用来恢复项目。
- 文件都在浏览器本地处理，但刷新或关闭页面前仍应导出工作簿和工作区 JSON 留存。
- 页面顶部的文件夹脚本只按编号创建空文件夹，不会整理或移动证据文件。
