# 珠海皇帝开发规格

## 产品边界

`session-bill-check` 核对场次表与账单表的姓名人次。它不核对金额、日期、训练性质、课程内容或单价，也不把姓名去重为人数。

## 输入契约

- 场次表和账单表支持 `.xls`、`.xlsx`。
- 场次表按表头读取 `学员`、`教员`、`检查员` 三列；姓名按中文或英文逗号拆分，只清理首尾空格。
- 账单表只读取精确命名为 `全动`、`理论` 的工作表，并按 `姓名` 表头定位列。
- `账单`、`全动汇总`、`理论汇` 等其他工作表不进入核对。

## 核对规则

- 三类场次角色合并统计；同一姓名每出现一次计一人次。
- 两类账单工作表合并统计；同一姓名每出现一次计一人次。
- 英文字母大小写不敏感；除首尾空格外不做姓名纠错、别名或中英文映射。
- 状态固定为 `一致`、`场次多`、`账单多`、`仅场次有`、`仅账单有`。

## 输出契约

- 页面展示输入识别摘要、状态统计、分布图、可筛选的姓名差异明细和选中姓名的两侧来源。
- 导出 Excel 固定包含 `核对汇总`、`姓名差异明细`、`场次拆分明细`、`账单姓名明细`。
- 输入结构错误时应指出缺失表头或工作表，不静默改用固定列。

## 代码与验证

- 核对与导出规则：`src/tool/app/session-bill-check/logic.ts`
- 页面接线：`src/tool/app/session-bill-check/main.ts`、`view.ts`
- 测试：`tests/tool/session-bill-check/logic.test.ts`
