# ohmyflight 开源宣传页开发规格

## 产品定位

`sponsor-page` 是 ohmyflight 的开源项目宣传与贡献页面，面向希望了解项目立场、真实工具能力并参与协作的人。页面只展示已确认的项目事实、真实运行画面、源码可以证明的能力边界和仓库现有贡献入口，不新增工具能力，也不把宣传文案当作业务规则。

## 页面入口

- 页面路径：`public/sponsor/index.html`
- 工具首页入口由 `src/site-visibility.ts` 中的 `homepage.sponsorEntry` 控制。
- 页面不设置顶部导航。

## 页面主线

页面按“开源项目身份 -> 八荣八耻 -> 真实运行案例 -> 算法透明 -> 开放参与 -> 贡献人员”的顺序组织。

Hero 只使用一次“为什么是 ohmyflight？”作为主标题，说明项目是一组从真实工作困难中生长出来的开源工具，并提供 GitHub 仓库和页面内真实案例入口。Pages、TypeScript、Python、Vitest、静态本地优先和 MIT 徽标只用于展示已经确认的项目事实。

## 八荣八耻

- 以解决真实业务需求为荣，以制造虚假问题为耻。
- 以交付真正可用的结果为荣，以只做表面文章为耻。
- 以事实、样例和复现为荣，以坑蒙拐骗和夸大宣传为耻。
- 以测试守住业务规则为荣，以能跑就算完成为耻。
- 以诚实说明能力边界为荣，以冒充万能工具为耻。
- 以减少重复劳动为荣，以为了炫技增加流程为耻。
- 以百倍提升生产力为荣，以效率提高就多接活为耻。
- 以开放报告、测试和优化为荣，以拒绝质疑、闭门造车为耻。

“八荣八耻”标题居中。八组荣耻对照使用“左侧八荣、中间龙颜大悦、右侧八耻”的三列结构，中央直接引用工具集合统一使用的 `public/tool/assets/status-done.png`，不复制或新增图片。颜色只区分“荣”与“耻”，不拆成八张卡片，不使用复杂流程线。

## 真实案例

- 猫咪：排在首位，展示一句指令处理文件任务，并提供带“猫咪官网”前缀的官网链接，以及全局安装、项目初始化和启动命令。
- 锁班皇帝：按人员、类型和日期批量录入非生产任务，并逐条保存成功、冲突和错误结果。
- 起落数查询助手：人工登录飞行门户后，按员工号和日期区间逐人查询飞行经历或左座经历与起落数，并把结果汇总到本地 Excel。
- 自动点 OA 助手：处理所有可直接确认的“已阅”待办；审批、意见、签署和特殊事项仍由人工完成。
- 培训皇帝：读取培训总表，按现有规则展示排班风险并导出新副本。
- 校对之王：用纯文字说明长文档插入内容后仍能保持后文对齐的实现链路，包括唯一锚点、最长递增子序列、滑动窗口、2-gram 相似度和最大权重链，并明确结果归集为参考新增、参考删除、内容修改和待确认。

五个 GIF 案例集中在统一的中性背景中，使用交替两栏和稳定间距，不再为每个案例设置不同的大块背景色。校对之王作为独立算法透明段落，不复制 owner 的 PPT 图片，不在网页中另建步骤栏、输入输出栏或流程线。

## 视觉与数据边界

- 使用 owner 提供并确认只含样例数据的现有 GIF 演示，不使用虚构界面，不新增或复制 PPT 图片。
- 站点保存独立 GIF 副本，不修改 owner 的原始 demo 文件。
- GIF 使用固定纵横比，避免加载时页面跳动；桌面和移动端必须保证标题、按钮和媒体不重叠。
- Hero、八荣八耻、案例总览、算法透明和开放参与使用白色、浅灰白和浅青白建立轻量全宽层级，不使用黑色大段、渐变、营销卡片阵列或无意义动画。
- Kitty 的三个命令使用中性命令框和深色复制按钮，复制成功后短暂显示“已复制”。
- 校对之王只描述源码和开发规格已经确认的文本对齐算法与结果边界，不使用“语义理解”“100% 准确”“毫秒级”等无法由当前实现和测试证明的表述。
- 前端最终视觉由 owner 人工确认。

## 开放参与

贡献区使用“开放，欢迎共同参与”说明协作立场，并提供贡献指南、Issues 和 Pull requests 三条路径。链接保持类似仓库列表的纵向结构，不使用营销卡片，不展示付款码或收款文案。

贡献人员区域显示当前发布人数，并用头像、姓名和具体贡献组成独立人员记录。名单集中维护在 `public/sponsor/contributors.js`，每项包含姓名、贡献说明、可选链接和可选头像链接；链接不限 GitHub，留空时姓名与头像不可点击。头像支持站内本地图片和 `http/https` 网络图片；未提供头像或头像加载失败时使用本地匿名 SVG。

`src/site-visibility.ts` 中的 `sponsorPage.contributors` 为 `false` 或名单为空时，页面显示配置的空状态，不渲染未确认人员。贡献人员标题、等待状态、空状态和发布人数模板统一维护在 `contributors.js` 的 `labels` 中；人数模板使用 `{count}` 占位符，HTML 不保存这些文案。
