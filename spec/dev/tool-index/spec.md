# 工具首页开发规格

工具首页统一展示公开工具入口，并通过“全部、重型、轻型、自动化、工作流”切换视图。首页默认显示“工作流”。

`src/tool/workflows-data.ts` 是工作流内容配置，只保存稳定 `id`、显示名称和工具 `entry` 顺序；节点名称、描述、图片和链接继续从正式工具清单生成。`src/site-visibility.ts` 的 `workflows` 映射独立控制每条工作流是否显示，开关键必须与工作流 `id` 一一对应。工具可以出现在普通分类和工作流中，每个节点仍独立打开对应工具页面。

当前工作流为：

- 锁班：文本拼接助手、姓名匹配员工号、锁班皇帝。
- 手册：PDF 加水印、校对之王、审计之王。
- 资质运行：酒店皇帝、重点人员标注、航线班次统计、飞行经历/左座经历起落数按天统计、技术等级运行资格查询助手。

工作流桌面端按从左到右的单向直线排列，窄屏按从上到下排列。每条工作流同一时刻只有一个慢速光点，箭头和光点方向一致；光点到达下一节点时，该节点轻微高亮。动画使用浏览器原生 Web Animations API，`prefers-reduced-motion` 下不显示流动光点和到达高亮。
