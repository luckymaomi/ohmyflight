# 自动点 OA 助手开发规格

## 产品边界

`oa-read-helper` 是可独立运行的 Python + Playwright APP，人工登录 OA 后循环处理可以直接确认的“已阅”待办。它不替用户审批、填写业务意见或绕过需要人工操作的特殊流程。

## 运行状态

- 启动时选择有限模式或无限模式；有限模式必须给出目标处理次数。
- 浏览器启动后由用户人工完成认证。
- 每轮定位当前待办、打开详情并尝试点击“已阅”，成功后返回或等待列表刷新。
- 普通条目失败时输出可见日志；督办等需要人工处理的类型由用户在浏览器完成后，脚本继续等待页面回到可处理状态。
- 无限模式持续运行直到用户中断或页面不可恢复；有限模式达到目标次数后退出。

## 输出与安全边界

- 输出仅为 OA 页面实际动作和终端日志，不生成业务文件。
- 日志统计本轮成功点击数量和需要人工处理或失败的条目。
- 自动化不得把“找到按钮”当作“已成功”，成功计数必须发生在页面动作确认之后。
- OA 页面结构或认证流程变化时允许失败并提示人工接管，不内置绕过方案。

- 当前无自动测试；验证依赖用户提供的页面日志、截图或内网实测结果。
