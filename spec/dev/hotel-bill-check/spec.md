# 酒店皇帝开发规格

## 产品边界

`hotel-bill-check` 比较酒店账单与入住登记记录，按同名和日期容差给每条账单寻找最近的登记记录。它不核对金额、房型或付款状态。

## 输入契约

- 两份输入均支持 `.xlsx`、`.xls`。
- 用户分别选择账单表与登记表的表头行、姓名列和日期列。
- 日期支持 Excel 日期、Excel serial 和常见年月日文本，统一到本地日历日比较。
- 登记表中的超链接证据随记录保留；同一行多个证据链接不能在导出时丢失。

## 匹配规则

- 候选必须姓名相同且日期差绝对值不超过用户选择的容差。
- 同一账单存在多个候选时选择日期差最小的记录。
- 登记记录第一次被使用标记为已匹配；后续账单再次使用同一登记记录时标记为重复匹配，供人工核对。
- 没有候选的账单标记为未匹配。
- 姓名不做别名、错别字或中英文映射。

## 输出契约

- 页面展示汇总、逐条状态、日期差和来源信息。
- Excel 导出包含核对结果及登记证据；多个链接展开为多个可点击的入住证明列。
- 工具不覆盖输入文件，所有解析和匹配在浏览器本地完成。
