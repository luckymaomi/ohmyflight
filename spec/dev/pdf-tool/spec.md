# PDF 工具开发规格

## 产品边界

`pdf-tool` 在浏览器本地提供页面提取、PDF 合并、PDF 转图片和图片转 PDF。工具不做 OCR、内容编辑、加密破解或版式重排。

## 页面提取

- 接受一份或多份 PDF，每份文件独立配置页码。
- 页码按从 `1` 开始的用户口径解析，支持单页和范围组合；越界或无有效页时不生成结果。
- 使用 PDF-lib 按指定顺序复制页面，每份源文件分别下载提取结果。

## PDF 合并

- 接受多份 PDF，用户可拖拽调整文件顺序。
- 合并时按当前文件顺序、每份文件的原页序复制到一个新 PDF。
- 输入少于一份或文件解析失败时不生成空结果。

## PDF 转图片

- 接受一份或多份 PDF，使用 PDF.js 渲染每一页。
- 用户选择图片格式与缩放倍率；渲染尺寸随倍率变化。
- 单个或多个结果按页面生成图片；批量结果使用 JSZip 打包下载。
- 只渲染页面视觉结果，不依赖 PDF 文字层。

## 图片转 PDF

- 接受多张浏览器可解码图片，用户可拖拽排序。
- 每张图片生成一页，按图片固有宽高设置页面并嵌入图像。
- 支持的实际格式由浏览器解码能力与 PDF-lib 嵌入路径共同决定；失败文件必须报错，不能静默丢页。

## 共同边界

- 文件只在浏览器内存中处理，不上传服务器，也不覆盖源文件。
- 大文件、高倍率或大量页面会明显增加内存和处理时间。
- 下载文件名由源文件基础名和操作后缀组成。

## 代码与验证

- 页面：`public/tool/app/pdf-tool/index.html`
- 模式逻辑：`src/tool/app/pdf-tool/extract.ts`、`merge.ts`、`pdf-to-image.ts`、`image-to-pdf.ts`
- 公共逻辑与入口：`src/tool/app/pdf-tool/shared.ts`、`main.ts`
- 当前没有独立业务测试；修改页码、排序或打包规则时应补充 `tests/tool/pdf-tool/` 测试。
