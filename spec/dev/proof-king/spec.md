# 校对之王开发规格

## 产品定位

`proof-king` 在浏览器本地对齐“我的手册”和“参考手册”，识别参考新增、参考删除、内容修改和待确认的完整修订事件，并把两侧原文与位置交给人工复核。它不判断业务等价、法规符合性或哪一版正确。

## 输入契约

- 两侧都支持标准 `.docx` 和带文字层的 `.pdf`，可任意组合。
- Word 读取全文；PDF 可限制页码范围，但每个原文单元不得跨页，切片继承真实页码。
- 明确噪音词条由独立 hook 配置；PDF 读取时只过滤规范化后与词条完全相等的独立文本项或整条视觉行，避免水印与同坐标正文拼接；包含相同文字的正常正文继续保留。
- 扫描 PDF、图片 PDF、OCR 和旧 `.doc` 不在范围。
- 文件与比对结果只在浏览器本地处理。

## 文本与对齐规则

- 展示副本保留完整原文；匹配副本只去除排版噪音，不删除数字或英文标识。
- 先按句末标点重建句子，过长句才按逗号、顿号、冒号继续切分。
- PDF 按坐标重建视觉行，并通过页面边缘位置和跨页重复率通用过滤页眉页脚。
- 唯一完全一致长句建立顺序锚点；锚点区间内召回单片段与相邻窗口。
- 对连续短行、表格行和扁平列表支持受限的 1:N、N:1、N:M 对应，最终链必须保持顺序且不得重复占用原文。
- 连续无对应片段聚合为新增或删除；可靠对应中的文字增删替换聚合为修改；证据不足保持待确认。
- 通用读取和对齐算法不内置部门、岗位、机型、法规、章节标题或示例数字特判。

## 事件与排序

- `参考新增`：参考侧有内容，我的手册在对应顺序区间无可靠对应。
- `参考删除`：我的手册有内容，参考侧在对应区间无可靠对应。
- `内容修改`：两侧可靠对应但原文存在逐字差异。
- `待确认`：同一顺序区间存在候选但证据不足。
- 一致内容只进入统计。结果固定按上述类别顺序排列，匹配率只是证据字段。
- 页面导航只生成事件视图，不改变比较结果。大类与关键词决定当前完整事件流；章节树从该事件流推导“章 -> 小节”计数和起点。点击章只展开或收起小节，点击小节只把完整事件流滚动到该小节起点，不过滤或移除其他事件。
- 同名小节只有在原文事件序列中相邻时才属于同一呈现组；搜索只在既有组内收窄结果，不能把被其他小节隔开的同名内容重新合并。未识别标题保留在 `未识别章节`、`未识别小节`，不得丢弃。
- 类别切换、章节和小节展开、关键词搜索均不修改事件类别、数量、原文位置或对齐证据。关键词结果显示命中侧与命中片段；类别计数同时保留匹配数和总数；事件流滚动时目录反向高亮当前小节。

## 人工决定与项目状态

- 每个修订事件独立保存 `待处理`、`纳入报告`、`不纳入` 三态人工决定；决定不写回 `RevisionEvent`，不参与对齐和分类。
- 当前大类与搜索范围支持批量纳入和重置；“只看纳入报告”是派生视图，不删除其他事件。
- 完整项目 ZIP 包含两本原手册、PDF 页码范围、比较快照、人工决定、视图状态和可读 Excel。项目清单记录工具类型、格式版本、文件路径、大小与 SHA-256。
- 导入必须先校验项目类型、版本、路径、文件哈希和业务状态，再读取两本原手册；全部成功后才替换当前页面项目。

## 原文复核

- 选中事件后并排展示两侧完整命中原文与前后上下文，逐字差异着色。
- 新增或删除缺失的一侧使用对齐阶段确认的附近共同锚点定位，不借用远处相同句或虚构上下文。
- 缺少共同前文或共同后文时显示文档边界或无可靠锚点说明，不用空白表示上下文状态。
- Word 使用固定高度连续阅读区并居中高亮；PDF 展示真实连续页，只渲染视口附近 canvas，远页及时释放。
- 两侧定位并行开始；事件列表虚拟滚动，只渲染可视窗口附近记录。

## 报告输出

Excel 报告固定包含 `总览`、`修订事件`、`参考新增`、`参考删除`、`内容修改`、`待确认`，并记录人工决定。每个事件保留类别、章节、小节编号、小节标题、组内序号、两侧位置、两侧完整原文、对应度、独有数字或英文标识和判断依据。Excel 与 Word 均可导出全部事件或仅导出已纳入事件。

Word 输出为标准 `.docx` 五列表格：章节、小节、对比说明、我的手册、参考手册。对比说明数据列保持空白，供人工填写。同类别、相邻且具有同一稳定小节标题的事件纵向合并章节和小节单元格，子事件的位置、原文和逐字差异仍各占一行；没有稳定小节标题的事件保持独立。章节和小节只在报告层整理，不进入对齐算法；我的手册删除文字标红，参考手册新增文字标蓝，相同文字保持黑色。

Excel 不使用物理合并单元格，按事件逐行保留章节、小节和组内序号，确保仍可筛选、排序和反向核对。
