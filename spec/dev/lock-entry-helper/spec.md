# 锁班皇帝开发规格

## 产品边界

`lock-entry-helper` 是飞行门户非生产任务录入自动化 APP。`app.py` 提供稳定单页串行流程，`superapp.py` 提供默认 20 页的实验性并发流程。网页只分发脚本和 Excel 模板；实际动作会向内网系统提交生产数据。

## 输入契约

- 支持批量粘贴、手动单条和 Excel 导入；并发版以 Excel 批量为主。
- 每条记录包含员工号、姓名、锁班类型、开始日期、结束日期，可选统一备注。
- 白名单在提交前过滤，未入白名单的记录不得打开录入动作。
- 结束日期不得早于开始日期；无法解析的员工号、类型或日期必须留在输入错误记录中。

## 提交链路

- 用户先完成浏览器与本轮参数设置，再人工登录并进入非生产任务录入页。
- APP 逐条填写员工号、锁班类型、起止日期和备注，并触发 Vue 表单依赖的页面事件。
- 查询结果必须按当前员工、锁班类型和日期范围精确归属，不能把相邻人员或并发页面结果串写。
- 门户返回冲突时不强行提交；记录冲突列内容并归档当前记录。
- 不可重试的信息弹窗直接归档为错误；可恢复页面异常按脚本既定策略处理。
- 并发版每个页面结果最终仍按原输入记录归档，页面完成顺序不能改变数据归属。

## 输出与恢复

- 两个 APP 都新建结果 Excel，每处理一条立即落盘，不回写输入 Excel。
- 结果表保留供人工复核的输入字段、处理状态、冲突与错误说明；成功记录不堆无用备注。
- 串行版另保存失败记录文本，方便复制后只重跑失败项。
- 中断后以结果文件中已落盘记录为完成证据，重跑前由用户排除已成功提交项，避免重复生产提交。

## 代码与验证

- 页面：`public/tool/app/lock-entry-helper/index.html`
- 串行 APP：`public/tool/app/lock-entry-helper/app.py`
- 并发 APP：`public/tool/app/lock-entry-helper/superapp.py`
- 页面壳：`src/tool/app/lock-entry-helper/shell.ts`
- 测试：`tests/tool/lock-entry-helper/result-matching.test.ts`，覆盖解析、白名单、门户结果归属、冲突和实时结果字段。
