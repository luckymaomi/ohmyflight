# 审计之王

## 功能

用于迎审时从检查单人工挑选关键词，并在多本公司手册中批量检索候选依据。

工具只做离线检索和定位，不做业务判断。

## 运行方式

- 启动站点后打开 `tool/app/audit-king/index.html`
- 页面在浏览器本地读取文件和检索，不上传到服务器

## 输入

- 1 份检查单 `.docx`
  只作为参考文本展示
- 多份公司手册 `.docx`
  作为被搜索的手册库
  支持多次追加上传，已上传手册可单本启用、停用或删除
- 人工维护的关键词
  可以从检查单参考文本中选中文字加入，也可以手动输入
  支持通过 Excel 导入和导出关键词池
- 文件夹脚本范围
  使用 `X.X-X.X` 格式填写，例如 `1.1-1.61`
  用于导出创建空文件夹的单文件 Python 脚本

## 搜索规则

- 支持一次性用多个关键词搜索多本手册
- 每个关键词单独统计命中数
- 支持查看全部关键词命中
- 支持点击某个关键词，只看该关键词命中
- 从检查单选中文字生成的关键词，点击关键词时会定位回检查单来源位置
- 关键词来源和关键词搜索相互独立；来源无法恢复时，关键词仍然参与手册搜索
- 关键词导入后会按检查单段落序号、来源坐标、来源文本和上下文尝试恢复检查单来源位置；不能唯一确认时不自动高亮
- 关键词来源无法恢复时，关键词池显示需人工确认；用户可在检查单参考中重新选中来源位置，并绑定到当前关键词
- 已绑定来源的关键词可以手动解绑；解绑只移除检查单来源定位，不影响关键词继续搜索手册
- 一个关键词可以人工绑定多条手册证据；手册证据只用于回到已确认过的候选依据位置，不自动进入审计篮子
- 手册证据导入后会按手册名称、手册 ID、段落序号、证据文本和上下文尝试恢复；不能唯一确认时不自动高亮或跳转
- 支持按某一本手册筛选命中
- 手册启用和停用控制搜索范围；停用手册不参与命中统计、命中列表或手册筛选下拉
- 支持精确搜索和宽松搜索
- 精确搜索按原文字面匹配
- 宽松搜索会忽略空白，并归一常见全角半角字符、中文英文括号和常见标点
- 搜索底层使用本地 FlexSearch 做候选段落召回，同时保留项目自己的 ngram 候选索引兜底
- 最终命中位置由项目自己的原文定位逻辑计算，用于页面高亮和 Excel 导出

## 关键词和来源定位设计

- 关键词是长期有效的检索资产
  关键词文字只用于搜索手册、统计命中和筛选命中。
- 关键词标记是人工备注字段
  用于标记该关键词对应的检查条款、检查项或工作分组，不参与手册搜索。
- 检查单来源是关键词的附属定位信息
  来源只用于在检查单参考里回到当时人工选中的位置。
- 关键词和检查单来源必须分离
  来源恢复失败时，不删除关键词，不停用关键词，不影响手册搜索。
- 关键词和手册证据必须分离
  关键词文字用于长期检索；手册证据是人工绑定过的候选依据位置，可以有多条。
- 检查单来源和手册证据必须分离
  检查单来源是 1 对 1，用于回到检查单；手册证据是 1 对多，用于回到手册。
- 手册证据和审计篮子必须分离
  手册证据是干活过程中的定位辅助；审计篮子是最终人工维护的条款和依据成果。
- 检查单参考只高亮人工选中的来源位置
  不按关键词文字在检查单全文里自动高亮所有相同词，避免把其它检查项误认为当前关键词来源。
- 绑定到当前关键词只更新来源
  不修改关键词文字、颜色、启用状态、命中数或手册搜索规则。
- 解绑当前关键词只删除来源
  不修改关键词文字、标记、颜色、启用状态、命中数或手册搜索规则。
- 绑定手册证据只把当前手册命中记录到当前关键词
  不修改检查单来源，不修改关键词文字，不修改审计篮子。
- 从全部详情选中原文加入手册证据只把选中原文记录到当前关键词
  不要求选中的原文包含关键词，不修改检查单来源，不修改审计篮子。
- 解绑手册证据只删除当前关键词下指定证据
  不删除关键词，不删除手册，不修改审计篮子。

## 来源状态

- 已定位
  当前关键词的来源能落到当前检查单中的有效段落和有效起止位置。
- 需人工确认
  关键词带有历史来源信息，但当前检查单中不能可靠恢复；关键词仍然可用于手册搜索。
- 无来源
  关键词由手动输入或无来源 Excel 导入产生，没有检查单来源定位。

## 来源恢复策略

导入关键词后，系统按以下顺序尝试恢复检查单来源：

- 先用当前检查单段落 ID、来源起点、来源终点和来源文本精确确认。
- 如果段落 ID 是旧版本导出的时间戳 ID，则从末尾 `-b序号` 提取检查单段落序号，再用段落序号、来源起点、来源终点和来源文本确认。
- 如果坐标已经移动，则用来源文本、来源前文、来源后文在同一段落中恢复。
- 如果同一段落无法恢复，则在整份检查单中用来源文本和上下文恢复。
- 只有唯一可信命中时才自动恢复。
- 找不到或多处命中时，不自动猜测，关键词池显示需人工确认。

## 关键词 Excel 结构

关键词导出包含两个 sheet：`关键词` 和 `手册证据`。

`关键词` sheet 字段：

- 序号
- 关键词
- 标记
- 启用
- 颜色
- 检查单段落
- 检查单段落序号
- 来源起点
- 来源终点
- 来源文本
- 来源前文
- 来源后文

`序号` 用于恢复关键词池显示和导出顺序，不参与手册搜索。`检查单段落` 是当前检查单稳定段落 ID。`检查单段落序号`、`来源文本`、`来源前文`、`来源后文` 用于检查单轻微调整后的恢复。

`手册证据` sheet 字段：

- 关键词序号
- 关键词
- 证据序号
- 证据来源
- 手册名称
- 手册ID
- 手册段落
- 手册段落序号
- 章节标题
- 证据起点
- 证据终点
- 全文起点
- 全文终点
- 证据文本
- 证据前文
- 证据后文
- 命中类型
- 备注

`手册证据` sheet 使用多行保存同一个关键词的多条证据，不把多条证据拼到 `关键词` sheet 的同一行。`证据来源` 取值为 `summary` 或 `selection`，分别表示从命中摘要加入或从全部详情选中原文加入。`手册名称`、`手册ID`、`手册段落`、`手册段落序号`、`全文起点`、`全文终点`、`证据文本`、`证据前文`、`证据后文` 用于手册重新上传后的定位恢复。

## 输出

- 检查单参考文本
  只高亮从检查单人工选中的来源位置；同一文字在检查单其它位置出现时不会自动高亮
- 手册解析状态
  显示每本手册名称、段落数量和启用状态，并支持启用、停用、删除单本手册
- 关键词池
  显示序号、关键词、标记、颜色、命中数、来源状态和手册证据数量，并支持手动输入序号排序、启用、停用、删除、导入、导出
- 当前关键词手册证据
  显示当前关键词已人工绑定的手册证据，区分摘要加入和全文选中，并支持解绑指定证据
- 命中摘要
  显示关键词、手册名、段落位置、命中类型和前后约 100 字上下文；选择具体关键词后，可以在摘要卡片上把该条命中加入或移出当前关键词的手册证据；已加入为手册证据的摘要必须在命中列表中直接突出显示
- 全部详情
  点击某条命中摘要后，默认显示同一本手册中围绕当前命中的约 2000 字连续原文，并只高亮当前命中；标题栏右侧显示当前已加载约多少字，并提供查看更多上下文按钮；点击后上下各扩约 1000 字；没有更多上下文时按钮禁用；切换关键词、筛选手册或切换摘要后恢复默认约 2000 字；选择具体关键词且当前详情属于该关键词时，可以在原文中选中一段内容并加入当前关键词的手册证据；已加入的全文选中手册证据在对应详情窗口内用绿色背景高亮原选区
- 审计篮子
  独立人工维护条款和依据内容，不绑定关键词、手册或命中详情
- 文件夹脚本导出
  页面可按 `X.X-X.X` 范围预览将要创建的文件夹名称，并导出一个独立 Python 文件
  默认 `1.1-1.61` 时，生成 `1.1-1.1`、`1.2-1.2` 这类文件夹名，范围内每个小编号对应一个空文件夹
  当前只支持同一一级编号内生成，例如 `1.1-1.61`
  导出的 Python 只使用标准库，只在运行脚本的当前目录创建缺失文件夹；已存在文件夹不报错；不会删除、移动或覆盖已有内容
- Excel 导出
  审计篮子支持导入和导出，列为条款名称、依据序号、依据内容、备注
  关键词池可单独导出为 Excel，包含 `关键词` 和 `手册证据` 两个 sheet

## 机械闭环

- 新建关键词
  用户从检查单选中文字或手动输入关键词。检查单选中生成的关键词会保存来源文本、段落序号、起止位置和前后文。
- 搜索手册
  关键词文字用于搜索多本手册；检查单来源不参与手册搜索。
- 绑定手册证据
  用户选择具体关键词后，可以在命中摘要卡片上把某一条命中加入为该关键词的手册证据，也可以在全部详情原文中选中一段内容加入为手册证据；一个关键词可以绑定多条手册证据。
- 导出关键词
  Excel 保存关键词本身、检查单来源字段和手册证据 sheet。
- 导入关键词
  关键词池被替换，先按 Excel 序号恢复显示顺序，无序号行按原行顺序排在后面；系统尝试恢复检查单来源和手册证据，并重新搜索当前已上传手册。
- 自动恢复
  能唯一确认来源时，检查单参考自动高亮来源位置。
- 人工确认
  来源无法可靠恢复时，关键词显示需人工确认；用户在检查单中重新选中来源位置后，点击绑定到当前关键词。
- 手动解绑
  用户可以清空当前关键词的检查单来源；解绑后该关键词显示无来源，但仍继续参与手册搜索和导出。
- 再次导出
  人工绑定后的检查单来源和手册证据写入 Excel，下一次导入继续使用长期结构恢复。

## 页面结构

- 第 1 行：关键词池和命中结果
- 第 2 行：检查单参考和全部详情
- 底部：当前关键词手册证据和审计篮子

## 手册范围规则

- 手册启用、停用是搜索范围控制
  停用后，该手册保留在手册列表中，但不参与重新搜索。
- 命中结果的手册筛选是展示过滤
  只在当前已启用手册的命中结果中切换视角，不改变手册启用状态。
- 停用当前正在筛选的手册时，筛选自动回到全部手册。
- 删除手册是移除输入材料
  删除后该手册从页面状态中移除；停用则可随时启用回来。
- 关键词 Excel 不记录手册启用状态
  关键词是长期资产，手册启停是当前页面会话里的搜索范围。

## 审计篮子规则

- 审计篮子和搜索结果相互独立
- 审计篮子和关键词手册证据相互独立
- 用户可以新增、编辑、删除条款名称
- 每个条款下可以新增、编辑、删除多条依据内容
- 新增条款时默认同时创建第一条空依据，页面滚动到审计篮子底部，方便立即填写
- 审计篮子支持按条款组手动输入序号排序；移动的是整个条款及其全部依据
- 条款序号输入为空、非数字或小于 1 时，页面恢复当前顺序，不执行移动
- 依据内容和备注都由用户人工填写
- 审计篮子不记录关键词、手册、段落位置或命中类型
- 审计篮子导入会替换当前审计篮子
- 审计篮子导入按条款名称归组

## 禁止行为

- 不自动拆检查项编号
- 不自动提取关键词
- 不自动生成符合性声明
- 不使用 AI、语义判断、向量检索或联网能力
- 不把检查单内容自动转成搜索任务
- 不把手册原文写入关键词导出文件
- 不把手册证据自动写入审计篮子
- 不把审计篮子和关键词手册证据强绑定
- 不在浏览器里执行导出的 Python
- 不让文件夹脚本删除、移动、重命名或覆盖任何已有文件
- 不用时间戳、随机数或会话态 ID 作为可导出来源定位的唯一依据
- 来源恢复不能唯一确认时，不自动选择第一个命中位置
- 手册证据恢复不能唯一确认时，不自动选择第一个命中位置

## 踩过的坑和最终决定

- 旧版本段落 ID 带时间戳
  同一检查单重新上传后，旧 Excel 的 `检查单段落` 无法匹配新页面段落，导致检查单来源不高亮。最终决定：文档和段落 ID 不再带时间戳。
- 只保存 `blockId/start/end` 不够长期稳定
  检查单轻微调整后，坐标可能移动。最终决定：关键词 Excel 同步保存段落序号、来源文本和前后文。
- 按关键词文字高亮整份检查单会误导
  同一个词可能出现在多个检查项里。最终决定：检查单只高亮人工选中的来源位置。
- 来源定位失败不代表关键词失效
  关键词仍然能搜索手册。最终决定：来源定位失败时显示需人工确认，而不是删除、停用或改写关键词。
- 自动恢复必须克制
  多处命中时系统不能替人判断。最终决定：只有唯一可信命中才自动恢复，其余交给人工绑定。
- 审计篮子不能被辅助定位数据污染
  关键词、检查单来源和手册证据都是为了加快人工查找；最终对外呈现的审计篮子必须由人维护。最终决定：手册证据只属于关键词，不自动进入审计篮子。
- 一对多证据不能塞进关键词单行
  一个关键词可能有多条手册证据。最终决定：关键词 Excel 保持 `关键词` sheet 记录关键词本身，另用 `手册证据` sheet 多行记录证据。

## 第一版不支持

- PDF 正文检索
- 图片 OCR
- `.doc` 旧 Word 格式
- Word 写回
- 跨浏览器会话保存

## 使用要点

- 检查单上传后只显示参考文本，用户需要自己选词或输入关键词
- 同一段检查单文字可以多次选取不同片段形成多个关键词
- 选中某个关键词后，再从检查单点击选中加入关键词，新关键词会插入到当前关键词下方；未选中具体关键词时追加到末尾
- 从检查单选中生成的关键词会保存来源文本和前后文，用于之后重新导入时恢复来源位置
- 绑定到当前关键词只更新检查单来源，不修改关键词文字、颜色、启用状态或手册命中规则
- 解绑当前关键词只清空检查单来源，不修改关键词文字、标记、颜色、启用状态或手册命中规则
- 绑定手册证据前需要先选中具体关键词；在查看全部命中状态下不直接绑定，避免证据归属不清
- 命中摘要卡片上的加入手册证据会保存该条摘要对应手册命中的位置、文本和上下文，不修改审计篮子
- 全部详情中的选中内容加入手册证据会保存选中原文、全文起止位置和上下文，不要求选中内容包含关键词，不修改审计篮子
- 全部详情上下文长度是临时阅读状态，不导入、不导出、不持久保存
- 一个关键词可以绑定多条手册证据；不需要的证据可以单独解绑
- 文件夹脚本导出是独立辅助功能，不读取检查单、关键词、手册证据或审计篮子
- 导出的 Python 文件需要放到目标目录运行；脚本会在当前运行目录创建空文件夹
- 修改关键词标记只更新人工备注，不修改关键词文字、检查单来源或手册命中规则
- 关键词池支持手动输入序号排序；排序只影响页面显示和关键词 Excel 导出顺序，不改变搜索规则或关键词内容
- 序号输入为空、非数字或小于 1 时，页面恢复当前顺序，不执行移动
- 关键词 Excel 导入时，序号小于 1 或非数字按无序号处理
- 搜不到的关键词会显示 0 命中，可以停用或删除
- 多次上传手册会追加到手册库，不会替换已有手册
