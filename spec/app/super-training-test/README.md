# 超级培训

## 功能

这是当前培训场景里最接近成品口径的工具，核心有五件事：

- 排班总览扫描
- 到期清单查询
- 一键更新有效期
- 一键预排班
- 培训计划核对

它面向的是“南货航新总表结构”，重点是直接处理一整本 merged workbook，而不是再拆“历史 sheet / 预排 sheet”。

## 运行方式

- 启动站点后打开 `tool/app/super-training-test/index.html`
- 这是浏览器本地工具

## 输入

### 通用输入

- 1 份培训总表 Excel
- 当前识别口径是：
  - `人员信息表`
  - `CRM`
  - 每个项目 1 张独立项目 sheet

### 排班总览

- 导入总培训表后直接扫描
- 默认评估日期区间是今天到下月底
- 可以手动修改评估开始日期和评估结束日期

### 到期清单查询

- 选择培训类型
- 选择开始月份
- 选择结束月份

### 一键更新有效期

- 选择项目
- 选择这些项目共同拥有的“已录入月份”

### 一键预排

- 选择项目
- 选择预排开始日期
- 选择预排结束日期

### 培训计划核对

- 选择项目
- 选择核对月份

## 输出

### 排班总览

- 页面默认只展示需要排班处理或人工确认的结果：
  - 已过期
  - 必须排
  - 推荐排
  - 异常
- 状态下拉可查看其他扫描结果：
  - 已排未录入
  - 已录入待更新
  - 正常
- 支持按项目、状态、到期月份和人员关键字筛选
- 支持导出当前筛选后的排班总览结果

### 到期清单查询

- 页面查询结果：
  - 项目
  - 员工号
  - 姓名
  - 有效期
  - 到期月份
  - 来源
- 支持导出查询结果
- 只读取人员信息表，不判断是否已排，不修改 Excel

### 一键更新有效期

- 页面预览：
  - 项目
  - 项目 sheet
  - 项目行号
  - 旧有效期
  - 新有效期
  - 判断
  - 处理结果
- 导出后的 Excel 副本
- 最前面会带 `更新报告`

### 一键预排

- 页面预排预览
- 导出后的 Excel 副本
- 新增 `项目名预排（生成）` sheet

### 培训计划核对

- 页面核对预览
- 导出后的 Excel 副本
- 直接修改原项目 sheet：
  - 已有可覆盖本轮到期安排的姓名标绿
  - 未覆盖本轮到期的姓名标红并补加
  - 补加行只写 `姓名` 和 `有效期`

## 输入输出样例

输入样例：

```text
项目：危险品
人员信息表中的旧有效期：2026-06-30
本次预排区间：2026-04-01 至 2026-04-30
```

输出样例：

```text
若命中预排窗口：
张三  旧有效期 2026-06-30  状态：命中窗口

若更新有效期成功：
张三  旧有效期 2026-06-30  新有效期 2028-06-29  处理结果：已更新
```

## 关键口径

### 总表结构

- `培训信息是否录入 = 是`
  这类记录只用于“一键更新有效期”
- `培训信息是否录入 != 是`
  这类记录只用于“一键预排”
- 如果备注包含 `取消`：
  - 未录入记录：这条计划记录无效，不参与覆盖判断
  - 已录入记录：数据矛盾，按异常提示

### 有效期更新

- 只看已录入记录
- 只有新有效期晚于今天，且不早于旧有效期时才回写
- 新生效锚点按“培训开始日期”算，不按培训结束日期算

### 预排

- 先调用统一排班评估核心。
- 只生成 `已过期`、`必须排`、`推荐排` 三类人员。
- `已排未录入`、`已录入待更新`、`正常` 不重复生成预排。
- 预排模板来源不是独立 sheet，而是同一项目 sheet 里的未录入行

### 排班总览

- 默认扫描区间是当天到本月底。
- 当前默认评估区间是当天到下月底。
- `已过期`、`必须排`、`推荐排` 均以评估日期区间为锚点判断。
- 扫描范围是人员信息表中的所有人员和所有可识别项目。
- 判断来源同时包含人员信息表当前有效期、项目 sheet 安排、培训规则和备注取消特判。
- 已排判断复用培训计划核对的覆盖判断。
- 默认视图只显示 `已过期`、`必须排`、`推荐排`、`异常`。
- `已排未录入`、`已录入待更新`、`正常` 默认隐藏，但可以通过状态筛选查看。
- 有窗口项目进入窗口后显示为 `推荐排`，到期月仍未排显示为 `必须排`。
- 无窗口项目提前 1 个月进入 `推荐排`，到期月仍未排显示为 `必须排`。
- 如果已找到覆盖安排，但没有 `培训信息是否录入 = 是`，显示为 `已排未录入`。
- 如果已找到覆盖本轮到期的已录入记录，但人员信息表仍是旧有效期，显示为 `已录入待更新`。
- 排班总览扫描和筛选不修改 Excel；导出只生成排班总览结果副本。
- 页面默认只展示简洁列：状态、项目、姓名、当前有效期、已排日期、说明。
- 导出 Excel 保留完整列：状态、项目、员工号、姓名、当前有效期、到期月份、最晚完成日期、已排日期、是否录入、来源、说明。

### 到期清单查询

- 查询来源只看 `人员信息表` 中的当前有效期。
- 按培训类型和月份范围筛选到期人员。
- 如果开始月份和结束月份相同，例如 `2026-04` 到 `2026-04`，只查询 2026 年 4 月到期人员。
- 如果开始月份和结束月份不同，例如 `2026-04` 到 `2026-05`，查询 2026 年 4 月至 2026 年 5 月到期人员。
- 不判断项目 sheet 里是否已经安排培训。
- 不修改原表。
- 有效期无法解析的记录进入异常提示。

### 培训计划核对

- 先看 `人员信息表` 里的当前有效期
- 以月为单位筛出“有效期落在所选月份”的人员
- 再看对应项目 sheet 里，是否已经有该人员一条能覆盖本轮到期的安排
- “已排”不再只看所选月份；只要在过期前已经排好，并且按项目规则能覆盖本轮到期，就算“已排”
- 如果找到了覆盖记录，但这些记录里没有 `培训信息是否录入 = 是` 的行，要在说明里注明：`未发现已录入信息`
- 备注包含 `取消` 的未录入记录不算已排覆盖
- `培训信息是否录入 = 是` 且备注包含 `取消` 的记录按异常提示
- 如果没有覆盖本轮到期的安排，就直接补到原项目 sheet
- 补加时只填 `姓名` 和 `有效期`
- 样式上：
  - 已排人员姓名标绿
  - 补加人员姓名标红

### 特殊规则

- `危险品`
  有效期按 2 年计算；复训窗口是“失效日前 3 个日历月至失效前 1 日”。
- `英语能力 / 汉语能力`
  英语能力按 `ICAO4` 算 3 年，汉语能力按 `ICAO5` 算 6 年；考试窗口是“失效日前 3 个日历月至失效日当天”。

## 使用要点

- 这是当前最值得优先写业务规范的培训工具
- 它不会直接覆盖原文件，而是先给预览，再导出副本
- 如果你的表还是“历史 / 预排分表”结构，这个工具不适合直接用
