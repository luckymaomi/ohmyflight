# 超级培训

## 功能

超级培训当前保留两个可见工作入口：

- 排班总览
- 有效期更新

它面向“南货航新总表结构”，直接导入一整本培训总表。页面不再提供独立的到期清单查询、预排班设置和培训计划核对入口。

## 运行方式

- 启动站点后打开 `tool/app/super-training-test/index.html`
- 这是浏览器本地工具

## 输入

- 1 份培训总表 Excel
- 当前识别口径是：
  - `人员信息表`
  - `CRM`
  - 每个项目 1 张独立项目 sheet

## 排班总览

- 导入总培训表后自动扫描。
- 默认评估日期区间是当天到下月底。
- 可以手动修改评估开始日期和评估结束日期。
- 扫描范围是人员信息表中的所有人员和所有可识别项目。
- 判断来源同时包含人员信息表当前有效期、项目 sheet 安排、培训规则和备注取消特判。
- 页面默认只展示需要排班处理或人工确认的结果：
  - 已过期
  - 已过期已排补训
  - 必须排
  - 已排未覆盖
  - 推荐排
  - 异常
- 状态下拉可查看其他扫描结果：
  - 已排未录入
  - 已录入待更新
  - 正常
- 支持按项目、状态、到期月份和人员关键字筛选。
- 支持导出当前筛选后的排班总览。
- 支持点击项目风险矩阵中的数字后，导出右侧人员明细。
- 排班总览扫描和筛选不修改 Excel。

## 有效期更新

- 选择人员信息表。
- 选择培训类型。
- 选择这些项目共同拥有的“已录入月份”。
- 点击“一键更新”生成预览。
- 导出后的 Excel 副本最前面会带 `更新报告`。
- 工具不会覆盖原文件。

## 关键口径

### 培训记录状态

- `培训信息是否录入 = 是`：用于有效期更新，也会被排班总览识别为已录入安排。
- `培训信息是否录入 != 是`：用于排班总览判断已排但未录入。
- 备注包含 `取消`：
  - 未录入记录：这条计划记录无效，不参与覆盖判断。
  - 已录入记录：数据矛盾，按异常提示。

### 排班总览判断

- 有窗口项目进入窗口后显示为 `推荐排`，到期区间内仍未排显示为 `必须排`。
- 无窗口项目提前 1 个月进入 `推荐排`，到期区间内仍未排显示为 `必须排`。
- 如果当前有效期已经早于评估开始日期，显示为 `已过期`。
- 如果已经过期但项目 sheet 里已有安排，显示为 `已过期已排补训`。
- 如果已有安排但不能覆盖本轮到期，显示为 `已排未覆盖`。
- 如果已找到覆盖安排，但没有 `培训信息是否录入 = 是`，显示为 `已排未录入`。
- 如果已找到覆盖本轮到期的已录入记录，但人员信息表仍是旧有效期，显示为 `已录入待更新`。

### 自动过滤

- 自动过滤清单维护在代码单文件中。
- 当前过滤事实：程春林、宋云龙在航空安保和英语能力中过期不用管。

## 输出

### 排班总览导出

完整列：

- 状态
- 项目
- 员工号
- 姓名
- 当前有效期
- 到期月份
- 最晚完成日期
- 已排日期
- 是否录入
- 来源
- 说明

### 右侧人员明细导出

完整列：

- 项目
- 状态
- 姓名
- 员工号
- 有效期截止日期
- 最晚完成日期
- 已排日期
- 是否录入
- 说明
- 来源

### 有效期更新导出

- 生成更新后的 Excel 副本。
- 添加 `更新报告`。
- 不覆盖原文件。

## 不适用

- 如果表还是“历史 / 预排分表”结构，这个工具不适合直接用。
- 当前页面不提供独立到期清单查询、预排班设置和培训计划核对入口。
