# 人员结构统计

## 功能定位

人员结构统计是纯前端工具。用户上传任意人员信息 Excel，工具按表头读取数据，按人员结构报告口径生成统计结果，并支持导出 Excel。

工具不绑定文件名、sheet 名或固定列位置。

## 输入

上传 `.xlsx` 或 `.xls` 文件。

默认读取用户选择的工作表。工具按表头识别字段。

必要表头：

- `姓名`
- `技术信息`
- `RAMA`
- `REUO`
- `RWAS`
- `EAMA`
- `EEUO`
- `EWAS`
- `原单位`
- `检查员资格`
- `行政职务`

可选表头：

- `员工号`
- `RSEA`
- `ESEA`
- `RANC`
- `RORD`
- `RJFK`
- `RLAX`
- `RNLU`

## 代码含义

单飞资格：

- `RAMA`：北美区域
- `REUO`：除俄罗斯外的欧洲区域
- `RWAS`：西亚和撒哈拉以北的非洲区域
- `RSEA`：东南亚及港澳台区域

英语通信/报务资格：

- `EAMA`：北美区域
- `EEUO`：除俄罗斯外的欧洲区域
- `EWAS`：西亚和撒哈拉以北的非洲区域
- `ESEA`：东南亚及港澳台区域

特殊/关注机场：

- `RANC`：安克雷奇
- `RORD`：芝加哥
- `RJFK`：纽约肯尼迪
- `RLAX`：洛杉矶
- `RNLU`：圣卢凯

当前报告结构里的航线资格和报务分类只直接使用美、欧、西亚三类组合：

- 单飞资格使用 `RAMA / REUO / RWAS`
- 报务资格使用 `EAMA / EEUO / EWAS`

## 技术信息规则

`技术信息` 冒号后的内容作为技术身份。

- 包含 `飞行教员`：教员
- 包含 `机长`，且不含 `飞行教员`、不含 `划转`：普通机长
- `划转机长`：在训机长
- 包含 `副驾驶`，且不含 `划转`：普通副驾驶
- `划转副驾驶`：转机型副驾驶

## 总人数和空地人员

- 空勤人员（已注册人员）：教员、普通机长、普通副驾驶。
- 地面人员：固定按 29 人统计。
- 总人数：空勤人员（已注册人员）加固定地面人员 29 人。

划转机长和划转副驾驶不属于已注册空勤人员，但在对应结构表里作为在训机长、转机型副驾驶统计。

## 管理人员

按 `行政职务` 判断：

- 管理人员：`行政职务` 非空
- 非管理人员：`行政职务` 为空

母数为结构统计人员，包含教员、普通机长、划转机长、普通副驾驶、划转副驾驶。

## 教员、机长、副驾驶

按结构统计人员统计，包含教员、普通机长、划转机长、普通副驾驶、划转副驾驶：

- 教员：飞行教员
- 机长：普通机长 + 划转机长
- 副驾驶：普通副驾驶 + 划转副驾驶

## 机长含以上各级别

母数为教员、普通机长、在训机长。

- 检查员：`检查员资格` 为 `公司检查员` 或 `委任代表`
- C/B/A 类教员：按 `技术信息`
- D/C/B/Z 类机长：按 `技术信息`
- 在训机长：`划转机长`

检查员是资格统计，不参与等级分类求和；等级分类本身应与母数闭合。若等级分类有漏项，输出为 `其他`，并列出员工号和姓名。

## 机长航线资格

母数为教员和普通机长，不含在训机长。

航线资格按 `RAMA / REUO / RWAS` 组合分类：

- 美+欧+西亚
- 美+欧
- 美+西亚
- 欧+西亚
- 仅北美带队
- 仅欧洲带队
- 仅西亚带队

航线机长：

- B 类及以上
- 没有 `RAMA / REUO / RWAS`
- 不是 Z 类机长

左座带飞：

- Z 类机长

机长航线资格占比不含在训机长。若教员和普通机长中有人不属于上述单飞资格、航线机长或左座带飞分类，输出为 `其他`，并列出员工号和姓名。

## 机长报务

母数为教员和普通机长，不含在训机长。

报务按 `EAMA / EEUO / EWAS` 组合分类：

- 美+欧+西亚
- 美+欧
- 美+西亚
- 欧+西亚
- 单美洲报务
- 单欧洲报务
- 单西亚报务
- 无报务

## 副驾驶级别

母数为普通副驾驶和转机型副驾驶。

- D/C/B 类副驾驶：按 `技术信息`
- A 类副驾驶：`A1类副驾驶` 或 `A2类副驾驶`
- 转机型副驾驶：`划转副驾驶`

若副驾驶级别分类有漏项，输出为 `其他`，并列出员工号和姓名。

## 副驾驶报务

母数为普通副驾驶，不含转机型副驾驶。

报务按 `EAMA / EEUO / EWAS` 组合分类：

- 美+欧+西亚
- 美+欧
- 美+西亚
- 欧+西亚
- 单美洲报务
- 单欧洲报务
- 单西亚报务
- 无报务

## 人员居住情况

按 `原单位` 判断：

- 本地：`原单位` 以 `总队` 开头，或等于 `777返聘`
- 异地：其他原单位

机长居住母数为教员、普通机长和在训机长。

副驾驶居住母数为普通副驾驶和转机型副驾驶。

## 原单位情况

母数为结构统计人员，包含教员、普通机长、划转机长、普通副驾驶、划转副驾驶。

映射规则：

- `总队777`、`777返聘` -> `飞行/总队 777`
- `总队737` -> `飞行/总队 737`
- `总队320` -> `飞行/总队 320`
- `总队909` -> `飞行/总队 909`
- `湖南分公司` -> `湖南`
- `湖北分公司` -> `湖北`
- `新疆分公司`、`新疆分公司（借）` -> `新疆`
- `河南分公司` -> `河南`
- `西安分公司` -> `西安`
- `重庆航空` -> `重庆`
- `汕头分公司` -> `汕头`
- `珠海分公司` -> `珠海`
- `广西分公司` -> `广西`
- `海南分公司` -> `海南`
- `上海分公司（借）` -> `上海`

未知原单位输出为 `其他`，并在口径中写明原始原单位；同时进入未识别数据，不静默丢弃。

## 输出

页面展示：

- 统计摘要
- 按人员结构报告顺序排列的统计表
- 需要核对的未识别数据

导出 Excel：

- `统计结果`
- `规则说明`
- `未识别数据`

输出不包含月份列，也不计算本月变化。工具只反映当前上传表的真实统计结果。

## Word 报告填充脚本

页面提供 `app.py` 下载。脚本属于人员结构统计的配套本地工具，不改变前端统计逻辑。

脚本输入：

- 人员信息 Excel
- 人员结构报告 `.docx`
- 可选月份参数 `--month 1-12`
- 可选工作表参数 `--sheet`
- 可选输出路径参数 `--output`
- 可选预演参数 `--dry-run`

不传 `--sheet` 时，脚本逐个工作表扫描表头，自动找到包含人员信息必要表头的工作表。

脚本输出：

- 生成新的 `.docx`，不覆盖原 Word 文件。

脚本写入范围：

- 写入 Word 前 9 张表的当前月份列、`本月变化`、`本月占比`。
- 写入到 `空勤人员原单位情况` 为止，后续人员引进情况和其他部门不处理。
- 写入前先清空目标月份列、`本月变化` 列、`本月占比` 列的正文单元格，再写入当前计算值，避免旧值残留。
- 不替换总人数段落，不替换飞行部人员结构标题段落。

Word 表格映射：

- 第 1 张表：`飞行管理人员占比`
- 第 2 张表：`教员、机长、副驾驶占比`
- 第 3 张表：`机长含以上各级别占比`
- 第 4 张表：`机长航线资格占比`
- 第 5 张表：`机长报务占比`
- 第 6 张表：`副驾驶级别占比`
- 第 7 张表：`副驾驶报务占比`
- 第 8 张表：`人员居住情况`
- 第 9 张表：`空勤人员原单位情况`

月份列规则：

- 传入 `--month` 时写入指定月份列。
- 不传 `--month` 时，自动识别前 9 张表中存在空值的月份列；如果无法唯一识别，则写最后一个月份列或提示用户指定月份。

本月变化规则：

- 当前月人数大于上月：`新增X人`
- 当前月人数小于上月：`减少X人`
- 当前月人数等于上月：`/`
- 上月为空或无法解析：`/`

脚本不自动插入新的 Word 行。当前报告模板中不存在的 `其他` 项只在控制台输出为需要核对，不强行改表结构。

脚本依赖：

```powershell
pip install openpyxl python-docx
```

常用命令：

```powershell
python app.py
python app.py --excel 人员信息.xlsx --docx 人员结构报告.docx --month 5
python app.py --excel 人员信息.xlsx --docx 人员结构报告.docx --month 5 --dry-run
```
