# ohmyflight 使用规范

这份目录只保留面向使用者的正式规范，不写实现细节，不展开技术方案。

这里的文档只回答这些问题：

- 这个工具是做什么的
- 什么时候适合用它
- 需要准备什么输入
- 最终会产出什么输出
- 使用时有哪些注意点

## 目录说明

- `spec/app/`
  每个正式工具各有 1 份独立规范
- `spec/tool-index.md`
  工具入口页与工作流占位页的界面规范
- `spec/app/README.md`
  所有工具规范的总索引

## 当前原则

- 只写用户需要知道的功能和规则
- 只写运行入口、输入、输出和样例
- 不写实现演进过程
- 不写实现结构
- 不写代码细节

## 建议阅读方式

1. 先看 `spec/app/README.md`
2. 再按工具名打开对应文档
3. 需要培训类工具时，优先看 `super-training-test.md` 和 `training-tools-suite.md`
