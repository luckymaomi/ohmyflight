# 工具入口页规范

## 页面目标

工具入口页集中展示当前全部正式工具，用户打开页面后可以直接搜索、筛选并进入工具。

## 页面结构

- `public/index.html` 只跳转到 `tool/index.html`，工具页是唯一维护的工具入口页。
- 页面只保留亮色展示，不提供明暗主题切换。
- 主页和开发者页使用与 skill 仓库一致的 `#f8fafc` 冷白页面底色，不使用偏米黄色背景。
- 页面不设置主标题、hero、宣传文案或操作教程。
- 页面继续使用仓库现有本地 Bootstrap CSS/JS，Skills 展开使用 Bootstrap Collapse。
- 顶部显示现有头像素材、固定文字“皇帝，您又来了”、搜索框、分类筛选和 GitHub 入口。
- 搜索框 placeholder 固定显示“恭喜发财”。
- 主页不直接展示提交记录、开发历史或 Skills；顶部“开发者”入口在新标签页打开独立开发者页。

## 工具入口

- 工具数据来自单一工具数组，名称、描述、状态、分类和入口均为显式数据。
- 工具入口地址由 `entry` 生成，并在新标签页打开。
- 工具按重型、轻型、自动化三类筛选；分类与搜索可以同时生效。
- 页面首次打开时默认选择“轻型”，用户仍可切换到全部、重型或自动化。
- 培训皇帝、审计之王、校对之王属于重型工具，自动执行类工具属于自动化工具，其余工具属于轻型工具。
- 分类只用于筛选，不在卡片内重复显示，也不改变卡片大小或排列规则。
- 所有工具使用同一网格和完全一致的卡片尺寸。
- 工具入口显示统一的“龙颜大悦”图片、名称和用途，不显示分类、状态点、打开箭头、序号或单字 Logo。
- 每张工具卡引用同一个本地图片资产，不为相同图片生成重复文件。
- 头像不添加圆形或彩色装饰边框，页面和卡片不使用彩色顶边、侧边线或机场、登机牌隐喻。
- 鼠标悬停工具卡时，真实卡片保持原位，通过阴影扩散形成缓慢的泡泡呼吸效果。
- 支持精细鼠标和悬停的设备上，鼠标连续停留在同一工具卡满 10 秒后，才从当前卡片位置生成独立覆盖层，在 5.5 秒内扩展到全屏并显示保持原比例的“龙颜大悦”图片；10 秒内移开不会显示，动画开始后移开会快速收回。
- 全屏效果不改变真实卡片尺寸、不推挤网格、不拦截点击，并在 `prefers-reduced-motion` 下禁用。

## 响应式与可访问性

- 宽屏显示六列工具，普通桌面、平板、窄屏和手机逐级收束为四列、三列、两列和单列。
- 页面不得产生整体横向滚动，长名称和描述必须在卡片内换行或截断。
- 主页和开发者页隐藏可见滚动条，但保留鼠标滚轮、触控板、触摸和键盘滚动能力。
- 分类按钮维护 `aria-pressed`，外链图标保留可访问名称。
- `/` 聚焦搜索框，`Escape` 清空当前搜索。
- 页面动画仅允许不改变布局的透明度淡入，并遵守 `prefers-reduced-motion`。

## 开发者页

- 开发者页使用 Bootstrap Tabs 分为 `Skills` 和 `提交记录`。
- 开发者页返回入口使用纯文字“返回”超链接，不显示箭头图标。
- Skills 数据在构建时直接读取 `.agents/skills/*/SKILL.md`，不维护第二份手工列表。
- 每项默认只显示 skill 名称和说明，点击后通过 Bootstrap Collapse 展开完整 `SKILL.md` 原文。
- 每项展开后提供“复制 Skill”按钮，优先使用 Clipboard API，失败时使用浏览器复制回退。
- skill 全文只在首次展开时写入页面，避免初始页面生成大量文本节点。
- 新增或删除仓库 skill 后，下一次构建自动同步开发者页。
- 提交记录读取构建生成的 `version.json`，显示当前构建提交、构建时间和 Git 提交列表。
- GitHub Pages 构建以完整 Git 历史检出仓库，部署后的提交记录必须包含当前分支完整历史，不能因浅克隆只显示最新一条。

## 交付要求

- 页面适配 GitHub Pages 静态部署。
- `public/.nojekyll` 随构建进入 `dist/.nojekyll`，明确要求 Pages 按静态文件发布，不进行 Jekyll 处理。
- 构建、类型检查和全量测试必须通过。
