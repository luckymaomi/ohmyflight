# 工具入口页规范

## 页面目标

工具入口页用于集中展示当前全部正式工具入口。

## 页面结构

- 首页 `public/index.html` 只跳转到 `tool/index.html`。
- 工具页是唯一维护的工具入口页，不维护两套工具列表。
- 页面只保留亮色展示，不提供明暗主题切换控件。
- 顶部只保留搜索框、GitHub 项目图标链接和源码下载图标链接，不设置标题区或 hero。

## 源码下载

- 页面顶部保留源码下载图标链接。
- 下载内容为当前仓库源码 zip。
- 打包方式为遍历整个仓库目录。
- 至少排除 `node_modules`、`dist`、`REF` 和打包产物本身。
- 构建时自动刷新 zip 文件并放入静态目录。

## 工具面板

- 工具入口页默认显示真实工具数据。
- 工具数据是单层工具数组，不维护分类分组。
- 工具名称与描述来自正式工具数据。
- 工具入口地址由 `entry` 生成，`entry` 是必填字段。
- 工具名称显示为蓝色下划线超链接，并在新标签页打开工具。
- 工具状态由 `status` 显式提供，只能是 `done` 或 `wip`，不能从工具名称或简介中的文字推断。
- `done` 显示完成图片，`wip` 显示文字 `开发中`。
- 表格列为 `工具 / 功能 / 状态`，三列平均分布。

## 交付要求

- 页面适配 GitHub Pages 静态部署。
- 构建、类型检查、测试必须全部通过。
