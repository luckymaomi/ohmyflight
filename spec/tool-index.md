# 工具入口页规范

## 页面目标

工具入口页用于集中展示当前全部正式工具入口。

## 顶部切换

- 顶部切换控件用于切换 `light` 与 `dark` 主题。
- 主题切换状态保存到浏览器本地存储。
- 重新打开页面时使用上次手动切换后的主题。
- 主题状态通过 `data-theme="light"` 或 `data-theme="dark"` 作用于页面。
- Bootstrap 页面同步使用 `data-bs-theme="light"` 或 `data-bs-theme="dark"`。

## 源码下载

- 页面左下角保留 `下载本站源码` 超链接。
- 下载内容为当前仓库源码 zip。
- 打包方式为遍历整个仓库目录。
- 至少排除 `node_modules`、`dist`、`REF` 和打包产物本身。
- 构建时自动刷新 zip 文件并放入静态目录。

## 工具面板

- 工具入口页默认显示真实工具数据。
- 分类只保留 `常用` 与 `其他`。
- `common` 归入 `常用`。
- 其余全部归入 `其他`。
- 工具名称与描述来自正式工具数据。

## 主题要求

- `light` 主题使用浅色背景、深色文字和清晰边框。
- `dark` 主题使用深灰背景、浅色文字和清晰边框。
- 主题样式不得使用透明毛玻璃效果。
- 文字、背景、边框、输入控件、表格和卡片必须保持可读对比度。

## 交付要求

- 页面适配 GitHub Pages 静态部署。
- 构建、类型检查、测试必须全部通过。
