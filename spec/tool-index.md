# 工具入口页规范

## 页面目标

工具入口页用于集中展示当前全部正式工具入口，并保留未来工作流入口的位置。

## 顶部切换

- 顶部只保留 `工具 / 工作流` 切换控件
- 切换控件负责在两个独立面板之间切换
- 不通过解释性标题说明当前状态
- 用户通过内容与深浅视觉直接感知当前状态

## 源码下载

- 页面左下角保留 `下载本站源码` 超链接
- 下载内容为当前仓库源码 zip
- 打包方式为遍历整个仓库目录
- 至少排除 `node_modules`、`dist`、`REF` 和打包产物本身
- 构建时自动刷新 zip 文件并放入静态目录

## 工具面板

- `工具` 为默认进入状态
- 工具面板显示真实工具数据
- 分类只保留 `常用` 与 `其他`
- 原 `common` 归入 `常用`
- 其余全部归入 `其他`
- 工具名称与描述必须来自正式工具数据

## 工作流面板

- `工作流` 面板当前不接入任何真实工作流内容
- 工作流面板只显示 `尚未接入`
- 工作流面板不得复用工具列表
- 工作流面板不得显示工具分类

## 状态隔离

- `工具` 与 `工作流` 必须是两个独立内容面板
- 切换到 `工作流` 时，不显示任何工具条目
- 切换到 `工具` 时，不显示 `尚未接入`
- 两个面板的内容不得相互混入

## 风格要求

- 整体风格贴近参考昼夜切换 demo
- 保持黑白、深浅主调
- 不增加搜索框
- 不增加额外介绍区
- 不增加与当前任务无关的说明

## 交付要求

- 页面适配 GitHub Pages 静态部署
- 构建、类型检查、测试必须全部通过
