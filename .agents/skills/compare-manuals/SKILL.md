# 手册差异比对

把任务理解为“发现修订事件”，不是计算一个相似度数字。搜索和相似度只召回候选，不直接证明业务等价。

## 工作流

1. 明确方向：哪一本是“我的手册”，哪一本是“参考手册”。参考手册有而我的手册没有，称为“参考新增”；反方向称为“参考删除/我的手册独有”。
2. 确认输入是标准 `.docx` 或带文字层的 `.pdf`。扫描件先说明不支持，不能用空文本继续判断。
3. 运行证据提取脚本：

   ```powershell
   python .agents/skills/compare-manuals/scripts/extract_manuals.py --my "我的手册.docx" --reference "参考手册.pdf" --output comparison-evidence.json
   ```

4. 读取 JSON 的 `documents`、`exact_anchors` 和 `gaps`。先看顺序稳定的唯一精确锚点，再逐个审阅锚点之间的缺口。
5. 对每个缺口允许 `1:1`、`1:N`、`N:1` 和 `N:M` 对应。先合并成完整条款或连续内容，再判断类别，禁止逐句孤立定性。
6. 输出报告，至少包含：类别、优先级、我的手册位置和完整原文、参考手册位置和完整原文、逐字变化摘要、判断依据、置信度。
7. 对低置信度内容标记“待人工确认”，不得为了减少待确认而强行匹配。

## 判断口径

- `一致`：去除排版、空白和标点差异后，内容相同；原文仍完整保留。
- `参考新增`：参考手册中存在一段连续实质内容，在相邻锚点限定的我的手册范围内没有可靠对应。
- `参考删除`：我的手册中存在连续实质内容，在参考手册对应范围内没有可靠对应。
- `内容修改`：前后锚点和核心文字证明是同一事项，但文字有增删替换。责任主体、数字、单位、等级、时间、否定词和条件变化都原样展示，不解释业务等价性。
- `待人工确认`：章节移动、改写幅度大、重复套话多或提取结构不足，无法可靠定性。

## 通用约束

- 不内置部门名、人员等级、机型、法规、章节标题或示例数字的业务特判。
- 不把组织关系、部门上下级或业务等价性写成算法事实，只报告原文变化。
- 标点用于先识别句子边界；去标点只发生在匹配副本中，不能破坏展示原文和位置映射。
- PDF 页眉页脚只能按跨页重复率和页面位置通用识别，不按具体文件文字过滤。
- 文档顺序是一等约束；一个文本单元不能被无限重复匹配。
- 匹配率不能替代新增、删除和修改结论，也不能作为报告主排序。

## 报告顺序

1. 参考新增
2. 参考删除
3. 包含数字、英文、等级、时间或否定变化的内容修改
4. 其他内容修改
5. 待人工确认
6. 一致内容统计

报告开头必须说明已确认事实、输入范围、提取限制和未验证内容。不要声称 AI 已判断业务合规性。

## 清理

证据 JSON 和临时报告默认放在用户指定位置或临时目录，不提交到仓库。任务结束后按用户要求保留最终报告，清理临时提取物。
