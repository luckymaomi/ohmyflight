## 页面结构

资料管理搜索框：role `textbox`，name `员工号或姓名简拼`。

查询按钮：role `button`，name `查询`。

员工链接：role `link`，name 为六位员工号，使用 `exact=True`。

个人弹窗关闭：`.pilotInfo-dialog-close`。

## HTML 网格事实

容器常见结构：

```text
container
  .hDiv table thead tr th
  .bDiv table tbody.list tr td
```

解析每行前先消费上一行仍有效的 `rowspan` 单元格；当前单元格的 `colspan` 要填入对应多个列。最终按表头长度补齐并 `dict(zip(headers, values))`，不能直接用每行 `td` 数量定位。

## 已确认责任名单案例

仅当用户明确要求该案例口径时使用：

- `Z类机长日期`：技术等级限定 `机型=777`；优先最早 `CAPZ` 生效时间，没有时取最早 `FOD2` 生效时间。
- `获取区域`：运行资格限定 `机型=777`、`类型=区域航线资格`、运行资格名称包含“单飞”。
- `实际日期`：命中资质的生效时间。
- 最近匹配：按单飞资质生效时间与源表发布日期的绝对天数差排序；同一最近日期多条时全部保留。
- `获取周期`：源表发布日期减 Z 类机长日期，按年月日和总天数展示。

这些条件必须放在任务级筛选函数或配置中，不能写死进通用门户表格抓取器。

## 已验证结果结构

`qualification-query-helper` 当前输出：

- `处理报告`：输入行号、员工号、输入姓名、页面姓名、匹配结果、两表条数、状态、说明、查询时间。
- `技术资料明细`：每条技术等级或运行资格一行，保留页面字段和抓取状态。
- `汇总`：输入、输出、总数、成功、失败、输入错误和中断状态。
- 同名文本报告：便于不打开 Excel 查看本次运行结果。
