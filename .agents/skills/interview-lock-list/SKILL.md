# Interview Lock List

在本仓库整理面试锁班名单时使用本 skill。

## 目标

把用户给的面试名单整理成锁班皇帝可导入的 Excel：

- Sheet `锁班数据`
- 列：`员工号`、`姓名`、`锁班类型`、`开始日期`、`结束日期`
- 另带 `类型列表`，优先从用户给的模板复制

当前面试锁班默认类型使用 `GRD-地面班`。面试名单不得默认生成 `GDO-地面休息`；如果 owner 明确给了其他当前有效类型，以 owner 当前要求为准。

## 工作流程

1. 先读取源 Excel 和模板 Excel 的 sheet、表头、前几行，不凭文件名或列号猜。
2. 确认模板结构：
   - `锁班数据` 表头是否为员工号、姓名、锁班类型、开始日期、结束日期。
   - `类型列表` 是否包含 `GRD-地面班`。
3. 解析面试表：
   - 常见结构是左侧日期列，右侧横向重复多组 `面试时间 / 姓名（员工号） / 面试类型`。
   - 日期如果只有日号，按用户当前语境或文件名确认年份和月份；不清楚时必须问 owner。
   - 从姓名单元格中提取 6 位员工号，姓名保留中文字符。
   - 支持 `姓名（123456）`、`姓名123456`、`姓名 123456` 这类格式。
4. 处理用户补充清单：
   - 常见格式：`员工号 姓名 日号`。
   - 追加到同一批 `锁班数据`。
   - 用 `员工号 + 日期` 去重。
5. 输出：
   - 新建文件，不覆盖源 Excel 和模板 Excel。
   - 建议命名为 `<源文件名>_plty.xlsx` 或 owner 指定名称。
   - 开始日期和结束日期通常同一天。
   - 按日期、员工号排序，便于人工核对。
6. 缺员工号处理：
   - 不要猜员工号。
   - 单独建 `待补员工号` sheet，列出日期、姓名原文、面试类型。
7. 验证：
   - 读回输出文件，汇报 sheet 名、总记录数、前几行、最后一行、缺员工号人数。
   - 检查 `锁班类型` 是否全为 `GRD-地面班`。
   - 检查是否存在空员工号、非 6 位员工号、空日期。

## 编码注意

- PowerShell here-string 或管道里的中文字面量可能变成问号。
- 写临时 Python 脚本时，涉及中文 sheet 名、表头、姓名和锁班类型，优先：
  - 使用 UTF-8 脚本文件，或
  - 用 Unicode 转义，或
  - 用 workbook index 读取模板 sheet。
- 不要因为终端显示问号就判断 Excel 文件损坏。

## 不做

- 不把生成的业务 Excel 提交进 git。
- 不删除用户原始 Excel，除非 owner 明确要求。
- 不为无员工号人员补猜员工号。
- 不把面试类型映射成锁班类型，除非 owner 明确给出口径。
